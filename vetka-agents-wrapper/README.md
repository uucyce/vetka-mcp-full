## What is this?

VETKA Agents provides:

- **Role-based agent system** — create custom agent roles with worktree isolation
- **Agent implementations** — ARC solver, Hostess, Learner, Architect, QA, PM and more
- **Worktree management** — agents work on separate git worktrees for parallel tasks
- **Role generation** — `add_role.sh` script for creating new agent identities

## Quick Start

### Create a new agent role

```bash
# Clone
git clone https://github.com/danilagoleen/vetka-agents.git
cd vetka-agents

# Install dependencies
pip install -r requirements.txt

# Run role generator
./add_role.sh \
  --callsign Alpha \
  --domain engine \
  --worktree alpha-engine \
  --tool-type claude_code \
  --model-tier claude-opus-4 \
  --role-title "Alpha Engineer"
```

This creates:
- Git branch `claude/alpha-engine`
- Worktree at `.claude/worktrees/alpha-engine`
- **`CLAUDE.md`** — Claude-specific instructions (read by Claude Code/Desktop)
- **`AGENTS.md`** — Generic agent protocol (read by any agent)
- Registry entry in `data/templates/agent_registry.yaml`

### What gets generated?

| File | Purpose | Generated by |
|------|---------|--------------|
| `CLAUDE.md` | Claude Code/Desktop instructions | `add_role.sh` + `generate_claude_md.py` |
| `AGENTS.md` | Generic agent protocol | `add_role.sh` + `generate_agents_md.py` |
| `AGENTS.md` (repo root) | Template/example | This repository |

## Agent Lifecycle

```
1. Create Role ──▶ 2. Init Session ──▶ 3. Claim Task
       │                  │                   │
       ▼                  ▼                   ▼
add_role.sh ──▶ vetka_session_init ──▶ vetka_task_board
                       │                   │
                       ▼                   ▼
              Memory Stack ◀────── Execute & Debrief
```

## Agent Protocol

1. **Init**: Call `vetka_session_init role=<callsign>`
2. **Task Board**: Call `vetka_task_board action=list`
3. **Claim**: Call `vetka_task_board action=claim task_id=<id>`
4. **Execute**: Work on the task
5. **Close**: Answer 3 questions:
   - Q1: What bugs did you notice?
   - Q2: What unexpectedly worked?
   - Q3: What idea came to mind?
6. **Complete**: Call `vetka_task_board action=complete task_id=<id>`

See [AGENTS.md](AGENTS.md) for full protocol.
