## 📌 Что хотел сказать Теренс Тао

### Суть Graph Fourier Analysis в одном абзаце

Классический Фурье-анализ раскладывает сигнал на частоты (синусы/косинусы). **Graph Fourier Transform (GFT)** делает то же самое, но на графе: раскладывает "сигнал на узлах" (например, значения эмбеддингов, активность, knowledge level) на **собственные функции графового лапласиана**. Низкие "частоты" — это гладкие, медленно меняющиеся паттерны (глобальные сообщества). Высокие "частоты" — резкие переходы, аномалии, границы между кластерами.

> **Ключевая идея Тао:** Граф — это не просто набор узлов и рёбер. У него есть **спектр** (набор собственных значений лапласиана), и этот спектр рассказывает о структуре графа больше, чем любая визуализация.

## 🔬 Математический фундамент (коротко, для понимания)

### Graph Laplacian — ядро всего

Для графа G с N узлами:

```
L = D - A

где:
  A = матрица смежности (adjacency matrix)
  D = диагональная матрица степеней (degree matrix)
  L = лапласиан графа
```

Собственные значения L: `0 = λ₁ ≤ λ₂ ≤ ... ≤ λₙ`

| Собственное значение | Что означает | Применение в VETKA |
|---|---|---|
| **λ₁ = 0** | Всегда 0 (связность) | Базовая проверка |
| **λ₂ (algebraic connectivity / Fiedler value)** | Насколько "хорошо связан" граф | **Spectral gap** = метрика здоровья KG |
| **λ₂ ≈ 0** | Граф почти распадается на компоненты | ⚠️ Сигнал: "мёртвая зона" в архитектуре |
| **λ₂ >> 0** | Граф плотно связан | ✅ Здоровая структура |
| **Высокие λₖ** | Высокочастотные паттерны — резкие переходы | Аномалии, неожиданные связи |

### Graph Fourier Transform

Если `u₁, u₂, ..., uₙ` — собственные векторы L, то для любого "сигнала" `f` на узлах:

```
f̂(k) = <f, uₖ> = Σᵢ f(i) · uₖ(i)

f̂ — это "частотное представление" сигнала на графе
```

**Низкие частоты (малые k):** гладкие паттерны → сообщества, кластеры
**Высокие частоты (большие k):** резкие переходы → аномалии, границы

### Tool 1: `spectral_health_check` — Здоровье Knowledge Graph

```python
import numpy as np
from scipy.sparse.linalg import eigsh
from scipy.sparse import csr_matrix

class SpectralHealthTool:
    """
    Tool для Архитектора: оценка здоровья Knowledge Graph
    через спектральный анализ лапласиана.
    
    @phase: 134.6 (research → implementation)
    @used_by: Architect Agent, MCC Dashboard
    """
    
    def __init__(self, adjacency_matrix: csr_matrix):
        self.A = adjacency_matrix
        self.n = adjacency_matrix.shape[0]
        # Degree matrix
        degrees = np.array(adjacency_matrix.sum(axis=1)).flatten()
        self.D = csr_matrix(np.diag(degrees))
        # Laplacian
        self.L = self.D - self.A
        
    def compute_spectrum(self, k: int = 10) -> dict:
        """
        Вычислить первые k собственных значений лапласиана.
        
        Returns:
            {
                'eigenvalues': [...],
                'spectral_gap': float,  # λ₂ - λ₁
                'algebraic_connectivity': float,  # λ₂
                'health_score': float,  # 0.0-1.0
                'diagnosis': str
            }
        """
        k = min(k, self.n - 1)
        eigenvalues, eigenvectors = eigsh(
            self.L.astype(float), 
            k=k, 
            which='SM'  # smallest magnitude
        )
        eigenvalues = np.sort(np.real(eigenvalues))
        
        spectral_gap = float(eigenvalues[1]) if len(eigenvalues) > 1 else 0.0
        
        # Health score heuristic
        # Хороший граф: spectral_gap > 0.1, но не слишком большой
        if spectral_gap < 0.01:
            health = 0.2
            diagnosis = "CRITICAL: Граф почти распадается. Есть изолированные модули."
        elif spectral_gap < 0.1:
            health = 0.5
            diagnosis = "WARNING: Слабая связность. Рекомендуется проверить мосты."
        elif spectral_gap < 1.0:
            health = 0.9
            diagnosis = "HEALTHY: Хорошая структура сообществ с достаточной связностью."
        else:
            health = 0.7
            diagnosis = "DENSE: Слишком плотный граф. Возможно, нужна декомпозиция."
        
        return {
            'eigenvalues': eigenvalues.tolist(),
            'spectral_gap': spectral_gap,
            'algebraic_connectivity': spectral_gap,
            'health_score': health,
            'diagnosis': diagnosis,
            'num_near_zero': int(np.sum(eigenvalues < 0.001)),
            'connected_components_estimate': int(np.sum(eigenvalues < 0.001))
        }
```

### Tool 2: `spectral_communities` — Обнаружение сообществ

```python
class SpectralCommunitiesTool:
    """
    Spectral clustering на графе Knowledge Graph.
    Замена / дополнение к HDBSCAN для случаев,
    когда важна СТРУКТУРА ГРАФА, а не только эмбеддинги.
    
    Ключевое отличие от HDBSCAN:
    - HDBSCAN работает в пространстве эмбеддингов (768-dim)
    - Spectral clustering работает на ТОПОЛОГИИ графа
    - Комбинация даёт лучший результат
    """
    
    def find_communities(
        self, 
        laplacian: csr_matrix, 
        n_communities: int = None,
        auto_detect: bool = True
    ) -> dict:
        """
        Найти сообщества через собственные векторы Fiedler.
        
        Если auto_detect=True, количество сообществ определяется
        по "eigengap heuristic" (максимальный скачок между λₖ и λₖ₊₁).
        """
        # Вычисляем достаточно собственных значений
        k = min(20, laplacian.shape[0] - 1)
        eigenvalues, eigenvectors = eigsh(
            laplacian.astype(float), k=k, which='SM'
        )
        idx = np.argsort(eigenvalues)
        eigenvalues = eigenvalues[idx]
        eigenvectors = eigenvectors[:, idx]
        
        if auto_detect and n_communities is None:
            # Eigengap heuristic: ищем максимальный скачок
            gaps = np.diff(eigenvalues[1:])  # пропускаем λ₁=0
            n_communities = int(np.argmax(gaps) + 2)  # +2 потому что начали с λ₂
            n_communities = max(2, min(n_communities, 15))
        
        # Используем первые n_communities собственных векторов для кластеризации
        from sklearn.cluster import KMeans
        features = eigenvectors[:, 1:n_communities+1]  # пропускаем первый (тривиальный)
        
        kmeans = KMeans(n_clusters=n_communities, n_init=10, random_state=42)
        labels = kmeans.fit_predict(features)
        
        # Формируем сообщества
        communities = {}
        for node_idx, label in enumerate(labels):
            label_key = int(label)
            if label_key not in communities:
                communities[label_key] = []
            communities[label_key].append(node_idx)
        
        return {
            'n_communities': n_communities,
            'communities': communities,
            'eigengap_used': eigenvalues.tolist()[:n_communities+2],
            'sizes': {k: len(v) for k, v in communities.items()},
            # Discrepancy check (из пункта 5!)
            'balance_score': 1.0 - np.std([len(v) for v in communities.values()]) / 
                            np.mean([len(v) for v in communities.values()])
        }
```

### Tool 3: `spectral_anomaly_detector` — Поиск аномалий

```python
class SpectralAnomalyTool:
    """
    Обнаружение аномальных узлов через высокочастотные компоненты
    Graph Fourier Transform.
    
    Аномалия = узел, чей "сигнал" (embedding, activity, knowledge_level)
    имеет аномально высокую энергию на высоких частотах.
    
    Практическое значение для VETKA:
    - Файл, который "не на своём месте" в иерархии
    - Модуль с неожиданными зависимостями
    - Узел, нарушающий DAG-паттерн
    """
    
    def detect_anomalies(
        self,
        laplacian: csr_matrix,
        signal: np.ndarray,  # значение на каждом узле (напр., knowledge_level)
        threshold_percentile: float = 95.0
    ) -> dict:
        """
        Graph Fourier Transform → выделяем высокочастотную энергию → аномалии.
        """
        n = laplacian.shape[0]
        k = min(n - 1, 50)
        eigenvalues, eigenvectors = eigsh(
            laplacian.astype(float), k=k, which='SM'
        )
        idx = np.argsort(eigenvalues)
        eigenvalues = eigenvalues[idx]
        eigenvectors = eigenvectors[:, idx]
        
        # Graph Fourier Transform: проецируем сигнал на собственные векторы
        # f̂(k) = <f, uₖ>
        gft_coefficients = eigenvectors.T @ signal
        
        # Высокочастотная энергия на каждом узле
        # Берём верхнюю половину спектра
        high_freq_start = k // 2
        high_freq_components = eigenvectors[:, high_freq_start:] @ gft_coefficients[high_freq_start:]
        high_freq_energy = np.abs(high_freq_components)
        
        # Определяем порог
        threshold = np.percentile(high_freq_energy, threshold_percentile)
        anomaly_mask = high_freq_energy > threshold
        anomaly_indices = np.where(anomaly_mask)[0].tolist()
        
        return {
            'anomaly_indices': anomaly_indices,
            'anomaly_count': len(anomaly_indices),
            'threshold': float(threshold),
            'high_freq_energy': high_freq_energy.tolist(),
            'spectral_energy_distribution': {
                'low_freq': float(np.sum(gft_coefficients[:high_freq_start]**2)),
                'high_freq': float(np.sum(gft_coefficients[high_freq_start:]**2)),
                'ratio': float(
                    np.sum(gft_coefficients[high_freq_start:]**2) / 
                    (np.sum(gft_coefficients[:high_freq_start]**2) + 1e-10)
                )
            }
        }
```

### Tool 4: `laplacian_embedding` — Замена UMAP для 3D Viewport

```python
class LaplacianEmbeddingTool:
    """
    Laplacian Eigenmaps как АЛЬТЕРНАТИВА UMAP для 3D визуализации в VETKA.
    
    Почему это может быть лучше UMAP для Knowledge Graph:
    
    1. UMAP работает в пространстве эмбеддингов (768-dim) и строит
       приближённый граф k-NN → теряет РЕАЛЬНУЮ топологию
    2. Laplacian Eigenmaps работают на РЕАЛЬНОМ графе KG → сохраняют
       НАСТОЯЩИЕ связи (imports, temporal, semantic)
    3. Для VETKA 3D viewport это означает: файлы, связанные
       реальными зависимостями, будут рядом в 3D пространстве
    
    @connects_to: client/src/components/canvas/ (Scene, FileCard, Edge)
    """
    
    def compute_3d_layout(
        self,
        laplacian: csr_matrix,
        normalize: bool = True
    ) -> np.ndarray:
        """
        Вычислить 3D координаты через Laplacian Eigenmaps.
        
        Returns:
            np.ndarray shape (N, 3) — координаты для 3D viewport
        """
        n = laplacian.shape[0]
        
        if normalize:
            # Normalized Laplacian: L_norm = D^(-1/2) L D^(-1/2)
            degrees = np.array(laplacian.diagonal())
            d_inv_sqrt = np.where(degrees > 0, 1.0 / np.sqrt(degrees), 0.0)
            D_inv_sqrt = csr_matrix(np.diag(d_inv_sqrt))
            L_norm = D_inv_sqrt @ laplacian @ D_inv_sqrt
        else:
            L_norm = laplacian
        
        # Берём собственные векторы 2, 3, 4 (пропускаем тривиальный u₁)
        eigenvalues, eigenvectors = eigsh(
            L_norm.astype(float), k=4, which='SM'
        )
        idx = np.argsort(eigenvalues)
        eigenvectors = eigenvectors[:, idx]
        
        # Координаты: u₂, u₃, u₄
        coords_3d = eigenvectors[:, 1:4]  # пропускаем u₁
        
        # Масштабирование для viewport
        coords_3d = coords_3d / (np.max(np.abs(coords_3d)) + 1e-10)
        coords_3d *= 10.0  # масштаб для Three.js scene
        
        return coords_3d
```

## 📊 Laplacian Eigenmaps vs UMAP — когда что использовать в VETKA

| Критерий | UMAP | Laplacian Eigenmaps |
|---|---|---|
| **Входные данные** | Эмбеддинги (768-dim) | Граф (adjacency matrix) |
| **Что сохраняет** | Приблизительные расстояния в embedding space | **Реальную топологию** графа |
| **Скорость** | Быстрый (O(N log N)) | Медленнее (eigsh ~ O(N·k²)) |
| **Когда лучше для VETKA** | Поиск семантических кластеров (похожий контент) | Визуализация **архитектурной структуры** (кто от кого зависит) |
| **3D Viewport** | Семантическое облако | **Архитектурная карта** |
| **Рекомендация** | Для поиска и discovery | Для Directed Mode / DAG view |

### Гибридный подход для VETKA 3D Canvas:

```
Режим "Semantic Cloud" (UMAP):
  → Файлы группируются по СМЫСЛУ
  → Хорошо для "найди похожее"

Режим "Architecture Map" (Laplacian Eigenmaps):
  → Файлы группируются по ЗАВИСИМОСТЯМ
  → Хорошо для "покажи структуру"

Режим "Hybrid" (взвешенная комбинация):
  coords = α * umap_coords + (1-α) * laplacian_coords
  → α контролируется ползунком в MCC
```

## 💡 Ключевой инсайт

Пункты 5 и 6 Тао **работают вместе**:

- **Пункт 5 (Discrepancy)** → **как строить** сбалансированный DAG (balanced layering в Sugiyama)
- **Пункт 6 (Fourier/Spectral)** → **как проверять** что DAG здоров и находить проблемы

Это как **строитель** (discrepancy) и **инспектор** (spectral):

```
Architect строит DAG → Discrepancy tools: "сбалансируй слои"
                                    │
                                    ▼
                        Spectral tools: "проверь здоровье"
                                    │
                                    ▼
                        JEPA: "предскажи, что сломается"
                                    │
                                    ▼
                        Human: "утверждаю / правлю"
```

> **Тао по сути дал нам полный инструментарий: от построения (discrepancy) через диагностику (spectral) до масштабирования (graphons из пункта 4). Это не отдельные идеи — это единый pipeline для архитектурного агента.**
