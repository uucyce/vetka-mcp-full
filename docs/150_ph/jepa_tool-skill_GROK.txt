### Почему JEPA как tool/skill имеет смысл (за/против)
JEPA (Joint Embedding Predictive Architecture) — это не просто эмбеддер, а predictive модель, которая учится предсказывать "missing parts" в latent space (из работ Meta 2023–2025: V-JEPA для видео, VL-JEPA для multimodal). Создание JEPA как **открытого инструмента** (tool/skill) в вашем agent_pipeline позволит:

- интегрировать в Cursor-style стриминг с approve/edit/reject,
- получить данные для A/B-тестов (algo-first vs JEPA-conditional).

### Почему это имеет высокий смысл именно сейчас
Ваши внутренние замеры (200–500+ мс на JEPA-path) — это уже реальная картина, а открытые источники (по состоянию на март 2026) дают только косвенные подтверждения:

- VL-JEPA (Meta, 2025) → selective decoding даёт **2.85×** снижение операций декодирования, inference в embedding-пространстве (non-autoregressive) → значительно ниже latency, чем у классических VLMs (например, ~24 мс vs 86 мс на update в одном из примеров для видео).
- Eidos (time-series JEPA-вариант, 2026) → inference ~6.71 мс на A100 (batch=1, context=512) — очень быстро для compact-моделей.
- MTS-JEPA и другие → подчёркивают scaling latency vs batch/window, но gap с PatchTST небольшой при больших батчах.
- Общий нарратив → JEPA-подходы позиционируются как **low-latency** для real-time (роботика, streaming, video), особенно когда не требуется полная генерация токенов.

Но **публичных чисел именно в мс на ваш сценарий** (text/code + pinned docs + overlay в чате) нет — всё измеряется на video/time-series или в относительных speedup'ах. Поэтому **только ваши логи** дадут правду.

### Как это реализовать (минимальный, но полезный план)

1. **Создать JEPA-tool** (в `mcp/tools/jepa_tools.py` или аналог)
   ```python
   class JEPA_predictive_embed(Tool):
       name = "jepa_semantic_core"
       description = "Генерирует predictive semantic core из чанков (текст/код/multimodal). Используется при high entropy / overflow для сохранения сути."
       args_schema = {
           "chunks": list[str],           # список чанков контекста
           "query": str,                  # опциональный текущий запрос
           "modality": str = "text",      # text / code / multimodal
           "max_chunks": int = 50
       }

       async def _run(self, chunks: list[str], query: str = "", modality: str = "text", max_chunks: int = 50):
           # вызов вашего существующего адаптера
           from src.services.mcc_jepa_adapter import embed_texts_for_overlay
           selected = chunks[:max_chunks]  # или ранжируем CAM/MGC
           embeddings, semantic_core = await embed_texts_for_overlay(selected, query=query)
           return {
               "semantic_core_desc": semantic_core.get("desc", ""),
               "compression_ratio": len("".join(chunks)) / len(semantic_core_desc) if semantic_core_desc else 1.0,
               "latency_ms": ...  # замерить здесь
           }
   ```

2. **Интегрировать в agent_pipeline с замерами**
   В `_stream_execution` добавить опциональный вызов:
   ```python
   start = time.perf_counter()
   jepa_result = await self.tools["jepa_semantic_core"]._run(chunks=relevant_chunks, query=task)
   latency_ms = (time.perf_counter() - start) * 1000

   yield await self._stream_event("tool_call_result", {
       "tool_name": "jepa_semantic_core",
       "tool_result": jepa_result,
       "latency_ms": round(latency_ms, 1),
       "trigger_reason": "entropy > 2.5"  # или что сработало
   })
   ```

3. **Калибровка триггеров и hysteresis — на реальных данных**
   Собирать метрики по 100–300 запросам (разные сценарии):
   - token_pressure, entropy (Shannon на salience scores), docs_count, modality_mix
   - JEPA latency (p50/p95), quality (cosine sim semantic_core vs full context)
   - compression ratio, first-token time с/без JEPA

   Затем подогнать:
   ```python
   # Пример после сбора логов
   if latency_p95 > 800:          # жёсткий лимит SLO
       JEPA_TRIGGER_THRESHOLD_ENTROPY = 3.0  # ужесточить
   HYSTERESIS_ON = 4
   HYSTERESIS_OFF = 3
   ```

4. **A/B и observability**
   - Feature flag `JEPA_TOOL_ENABLED`
   - Trace: `state.trace['jepa_latency_ms']`, `jepa_triggered`, `compression_before/after`
   - Сценарии для теста: text-only chat (ожидаем редко), 20+ pinned docs (часто), multimodal viewport (очень часто)
