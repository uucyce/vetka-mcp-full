## 📊 Диагностика по векторам (из 3D_PERFORMANCE_AUDIT.md + layout audits)

**Текущие метрики (из audit):**
- **FPS**: <30 на 2000+ нод (цель 60).
- **Draw calls**: 2000+ (по 1 на FileCard).
- **Re-renders**: 30-50% избыточно (нет React.memo).
- **Память**: ~1MB/текстура × 2000 = 2GB+.
- **Layout**: Backend (fan_layout.py) даёт **вертикальное дерево** (root Y=0, дети Y+=Y_PER_DEPTH вверх), но frontend **перезаписывает** в `calculateSimpleLayout()` (Y=depth*20 — горизонтально плоско).

**Корень лагов (ТОЧНО из audit):**
1. **No InstancedMesh** (`FileCard.tsx`): 2000 meshes вместо 1 InstancedMesh.
2. **No Frustum Culling** (`App.tsx:310-326`): Рендерит **все** ноды.
3. **Texture Regen** (`FileCard.tsx:362-692`): Canvas на каждый state change (14 deps).
4. **useFrame per node** (`FileCard.tsx:233-278`): LOD/dist каждый кадр ×2000.
5. **No Memo** (`FileCard/Edge`): Re-render на parent changes.

**Layout баг (из GROK_RESEARCH_PROMPT_v2.md + VETKA_DAG_Layout_Analysis):**
- Backend **правильный**: `folder_y = parent_y + Y_PER_DEPTH` (вертикаль веером).
- Frontend **перезапись**: `useTreeData.ts:140` → `invalidRatio>0.5` → `layout.ts:38` (Y=depth*20).
- **Почему плоско**: Fallback срабатывает на "invalid" (0,0,0), даже если backend дал Y.

## 🎨 Твои идеи: Конкретная реализация

### 1. **Parallax + Z-depth + Folder sizing**
- **Z**: Папки Z=-50 (ближе), файлы Z=0 (дальше). LOD: папки всегда high-res, файлы low-poly на расстоянии.
```tsx
// FileCard.tsx: LOD по distance + type
const lodScale = isFolder ? 1.2 : Math.max(0.3, 1 / (distance / 500));
mesh.scale.setScalar(lodScale);
```
- **Размер по объёму**: `size = Math.sqrt(folderSizeKB / 1000) * baseSize` (из Qdrant payload или tree_routes.py).
- **Parallax**: CameraController.tsx — mouse parallax для hover (shift X/Y на 10-20px).

### 2. **Сворачивание/Виртуализация (Browser-like lazy load)**
- **Virtualization**: react-window + frustum (audit rec #2). Грузим **только viewport** (15/1575 видно сейчас).
- **Collapse**: Folder icon → клик → анимация scaleZ 0 + emit 'folder_collapsed' (useSocket.ts).
  - Объём: Badge "1.2MB / 33 files" (из folder metadata).
  - Клик: Вылетают файлы (spring anim + highlight).
```tsx
// TreeEdges.tsx: Только visible edges
const visibleEdges = edges.filter(e => frustum.containsPoint(midpoint(e.start, e.end)));
```
- **Эффект**: Замена 33 файлов на папку = **-90% draw calls** (папка = 1 mesh).

## 3D vs 2D: **Hybrid — 3D primary + 2D fallback/mini**

**Вектора говорят: 3D держит** (с instancing/culling = 60FPS). Но:
- **2D Fallback**: При >2000 nodes или low perf → Canvas2D overview (SVG tree).
- **Hybrid**: 3D viewport + 2D HUD (search/minimap/timelines). react-window для list views.
- **Почему не full 2D**: Теряем immersion (parallax/Z/fly-through) — core VETKA фича.
