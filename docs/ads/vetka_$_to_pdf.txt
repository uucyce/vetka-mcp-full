## **Slide 3: What is VETKA? (Core Vision)**  
**Not a Tool – A New OS Paradigm**  
*(From codebase: `docs/Vetka_first_understand.txt` – "Montage of Reality", not video editing)*  

- **Spatial 3D Graphs**: Files/folders as nodes. Dependencies (imports, temporal, semantic) as edges. Y=Knowledge Level (raw data → derivatives).  
- **Multimodal**: Code (imports), Docs (links), Images/Videos (timestamps/metadata), Artifacts (scanned via `artifact_scanner.py`).  
- **Eternal Memory**: Qdrant/Weaviate triples + changelog. Never loses context across sessions/agents.  
- **Local-First**: Tauri desktop (`app/start_vetka.sh`). GPU-heavy modes → remote MCPs (`tree_routes.py`).  

**Demo Flow**: Scan codebase → 3D viewport (698 nodes visible) → Drag "tests/" → Auto-fan layout → Query agents on graph.  

**MYCELIUM**: Command Center layer. DAG pipelines + agent orchestration. "Run Flux on node X → Link output as child."
