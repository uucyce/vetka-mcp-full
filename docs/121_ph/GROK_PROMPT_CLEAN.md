## ЗАДАЧА 2: Chat-to-Tree Binding (ГЛАВНОЕ)

### Проблема
```
Текущее состояние:
- Чат это отдельная сущность
- 3D дерево это отдельная сущность
- Между ними нет связи!

Желаемое состояние:
- Один chat_id на весь разговор
- НО каждое сообщение привязано к своему узлу в дереве
- Визуально: сообщения "растут" из тех файлов о которых спрашивали
```

### Гибридная логика

**Правило 1:** `chat_id` = UUID, создаётся один раз, никогда не меняется

**Правило 2:** Каждая пара сообщений (Q+A) имеет `anchor_node_id`:
- Если спросили о файле → anchor = этот файл
- Если несколько файлов → anchor = первый файл ИЛИ родительская папка
- Если нет файлового контекста → anchor = позиция камеры (виртуальный узел)

**Правило 3:** Визуализация:
- В **Панели истории** (слева): один непрерывный чат
- В **Ленте чата**: бесшовный поток сообщений
- В **3D дереве**: пары сообщений как дочерние узлы anchor'а

### Вопросы
1. Алгоритм определения `anchor_node_id` из контента сообщения?
2. Как обрабатывать "camera position" anchors? (нет файла, только 3D координаты)
3. Message nodes — реальные узлы дерева или overlay decorations?
4. Как обрабатывать переименование/перемещение файлов посреди чата?

## ЗАДАЧА 4: Модели данных

### Предлагаемая модель сообщения
```python
class ChatMessage:
    id: str                    # UUID
    chat_id: str               # Parent chat UUID
    role: str                  # "user" | "assistant" | "system"
    content: str
    timestamp: datetime

    # Binding
    anchor_node_id: str        # К какому узлу привязано
    anchor_type: str           # "file" | "folder" | "camera" | "artifact"
    mentioned_files: List[str] # Все упомянутые файлы

    # Artifacts created
    artifacts: List[str]       # IDs артефактов

    # LLM metadata
    model: str
    tokens_in: int
    tokens_out: int
```

### Предлагаемая модель артефакта
```python
class Artifact:
    id: str
    type: str                  # "code" | "document" | "config"

    # Origin
    chat_id: str
    message_id: str
    creator_agent: str         # "architect" | "coder" | "user"

    # Content
    filepath: str              # Если сохранён
    content: str
    language: str              # "python", "typescript"

    # Status
    status: str                # "draft" | "pending" | "approved"

    # Tree binding
    target_node_id: str        # Для какого узла артефакт
```

**Вопросы:**
