================================================================================
                         TRUNCATION STRATEGY
================================================================================

Old Strategy (char-based):
─────────────────────────
  content[:3000] + "... [truncated]"

  Problem: Loses potentially important end-of-file content
           (exports, main functions, entry points)


New Strategy (token-based, head+tail):
──────────────────────────────────────
  max_chars = max_tokens * 4

  head = content[:max_chars * 0.6]    # 60% from beginning
  tail = content[-max_chars * 0.4:]   # 40% from end

  result = head + "\n\n... [truncated N chars] ...\n\n" + tail

  Benefit: Preserves imports (head) AND exports/main logic (tail)


Example (1000 token limit = 4000 chars):
────────────────────────────────────────
  Original file: 10000 chars

  Head: first 2400 chars (60%)
  Tail: last 1600 chars (40%)
  Truncated: 6000 chars (middle removed)

  Output:
  ┌────────────────────────────────────┐
  │ # imports, docstrings, class defs  │ ← 2400 chars
  ├────────────────────────────────────┤
  │ ... [truncated 6000 chars] ...     │
  ├────────────────────────────────────┤
  │ # exports, main(), __all__         │ ← 1600 chars
  └────────────────────────────────────┘
