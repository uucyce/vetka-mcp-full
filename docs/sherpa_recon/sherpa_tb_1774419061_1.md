Problem Summary
The bug report identifies a tolerance mismatch between frontend and backend keyframe detection:

At 120fps, the 20ms tolerance spans nearly 2.5 frames, which can cause:
False positives: Multiple keyframes detected within a single intended keyframe window
Sync drift: Frontend-selected keyframes may not match backend-processed keyframes
Timeline inconsistency: User sees one keyframe, backend processes another
Root Cause Hypothesis
The 20ms tolerance likely originated from:
Legacy debouncing for UI interactions (mouse/touch events)
Assumption of 30fps standard (where 20ms ≈ 0.6 frames, acceptable)
Not updated when high-frame-rate support was added
