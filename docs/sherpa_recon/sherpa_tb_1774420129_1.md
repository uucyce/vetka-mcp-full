## Research Response

[DEBRIEF-BUG] MenuBar Add Marker/Add Comment Marker – Fragile KeyboardEvent dispatch

Severity: Medium (UX regression, inconsistent behavior)
Affected Component: MenuBar (React), MarkerStore (Zustand/Redux)
Detected by: Cross-cutting agent observation
Related: MARKER_ZETA.F2, Alpha store actions

Problem Statement

The MenuBar’s “Add Marker” and “Add Comment Marker” actions currently trigger marker creation by synthesizing and dispatching a KeyboardEvent on document. This approach is fragile because:

It depends on the focus of the timeline/player panel – if the user has clicked elsewhere (e.g., property editor, file browser), the keyboard event is swallowed or ignored.

It bypasses the central MarkerStore, breaking predictability and making it impossible to add markers from outside the timeline component (e.g., from a keyboard shortcut manager, voice command, or debug panel).

It creates timing and race conditions because the actual marker creation logic lives inside the timeline component’s onKeyDown handler, which may not be synchronous with the MenuBar’s action.

Expected Behavior

Clicking “Add Marker” or “Add Comment Marker” in the MenuBar should:

Always create a marker at the current playhead position, regardless of which panel has focus.

Use a dedicated store action (e.g., useMarkerStore.getState().addMarker({ time, comment? })).

Be testable and decoupled from keyboard event simulation.

The timeline component listens for 'keydown' and calls addMarker only when the timeline container is focused.

Proposed Fix

Replace the KeyboardEvent dispatch with a direct call to a MarkerStore action that the Alpha module (or a shared store) exposes.

1. Define the MarkerStore action (Zustand example)

// stores/markerStore.ts
import { create } from 'zustand';

interface Marker {
  id: string;
  time: number;
  comment?: string;
}

interface MarkerStore {
  markers: Marker[];
  addMarker: (marker: Omit<Marker, 'id'>) => void;
  removeMarker: (id: string) => void;
}

export const useMarkerStore = create<MarkerStore>((set) => ({
  markers: [],
  addMarker: (marker) =>
    set((state) => ({
      markers: [...state.markers, { ...marker, id: crypto.randomUUID() }],
    })),
  removeMarker: (id) =>
    set((state) => ({
      markers: state.markers.filter((m) => m.id !== id),
    })),
}));
2. MenuBar uses the store directly

// MenuBar.tsx
import { useMarkerStore } from '../stores/markerStore';
import { usePlayerStore } from '../stores/playerStore'; // for current time

const handleAddMarker = () => {
  const currentTime = usePlayerStore.getState().currentTime;
  useMarkerStore.getState().addMarker({ time: currentTime });
};

const handleAddCommentMarker = () => {
  const currentTime = usePlayerStore.getState().currentTime;
  const comment = prompt('Enter marker comment:');
  if (comment) {
    useMarkerStore.getState().addMarker({ time: currentTime, comment });
  }
};
