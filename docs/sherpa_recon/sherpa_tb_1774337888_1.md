## Research Response

[DEBRIEF-BUG] scan_all() called on every session_init — adds ~200ms latency
Summary

scan_all() is being invoked unconditionally during every vetka_session_init call. This adds ~200ms overhead per session initialization. While currently acceptable, the cost may grow linearly as more tools/tasks are added to the system (O(n) scan over registry entries). The root cause is a missing cache/persistence layer for the aggregated tool/task index.

Affected component: src/mcp/tools/session_tools.py → vetka_session_init
Cross-cutting with: src/orchestration/task_board.py (likely where scan_all() resides) and src/services/agent_registry.py

Impact

Latency regression: Each MCP session now takes ~200ms longer to bootstrap.

Scalability risk: Adding 10x more tools/tasks → ~2s overhead per session.

Resource waste: Repeated I/O and computation for identical data across sessions (no sharing).

Root Cause

scan_all() is called inside the session initialization flow to gather:

All available MCP tools (names, schemas, permissions)

Task board tasks (open, claimed, completed)

Agent registry entries (branches → roles)

The current implementation does not cache the scan result. Because session state is short-lived and each agent (Claude Code, Desktop, Cursor) creates a fresh MCP session per chat/thread, the scan repeats unnecessarily.

Solution

Introduce a TTL-based cache for scan_all() results, similar to the existing _doc_cache in task_board_tools.py. The cache should:

Invalidate only when the underlying data changes (new tool registered, task board updated).

Use a short TTL (e.g., 5 seconds) to balance freshness vs. latency.

Optionally support forced refresh via a force_scan parameter for debugging.

Cache design:

Key	Value	TTL	Invalidation triggers
"scan_all:v1"	(tools_list, task_board_snapshot, agent_registry_snapshot, timestamp)	5 sec	Tool registration, task creation/completion, agent registry change

# src/mcp/cache/scan_cache.py
import time
from typing import Any, Dict, Optional

_scan_cache: Dict[str, tuple] = {}  # key → (result, timestamp)
_SCAN_CACHE_TTL = 5  # seconds

def get_cached_scan(force_refresh: bool = False) -> Optional[Any]:
    key = "scan_all:v1"
    if not force_refresh and key in _scan_cache:
        result, ts = _scan_cache[key]
        if time.time() - ts < _SCAN_CACHE_TTL:
            return result
    return None

def set_cached_scan(result: Any) -> None:
    key = "scan_all:v1"
    _scan_cache[key] = (result, time.time())
