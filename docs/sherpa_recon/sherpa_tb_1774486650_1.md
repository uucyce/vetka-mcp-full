🔍 Bug Analysis: INSERT OR REPLACE Data Loss in Task Board
Root Cause

INSERT OR REPLACE in SQLite deletes the existing row and inserts a new one. If the new row omits certain columns (e.g., created_at, commit_hash), those values are lost forever. This is not an update – it’s a destructive overwrite that drops any field not present in the INSERT statement.

Why it happens even with perfect IDs:
The conflict is on the PRIMARY KEY (e.g., id). REPLACE removes the old row, then inserts the new row. Any column not explicitly set in the INSERT receives its default value (NULL or empty). For example:

sql

Download
-- Existing task: id=42, title="Fix audio sync", status="done", created_at=1234567890
INSERT OR REPLACE INTO tasks (id, status) VALUES (42, 'pending');
-- After: title = NULL, created_at = NULL, status='pending' → all metadata lost!
Where is the Bug? (Likely in task_board.py)

Although your snippet doesn’t show the exact SQL, the pattern appears in methods that “upsert” a task, e.g. add_task(), update_task(), or save_task_board(). Look for:

🔧 The Fix: Use INSERT ... ON CONFLICT DO UPDATE (SQLite 3.24+)

Replace every INSERT OR REPLACE with proper upsert that preserves all columns.

def upsert_task(task: dict):
    conn.execute("""
        INSERT INTO tasks
        (id, title, status, assigned_to, updated_at, created_at, commit_hash, ...)
        VALUES (?, ?, ?, ?, ?, COALESCE(?, created_at), COALESCE(?, commit_hash), ...)
        ON CONFLICT(id) DO UPDATE SET
            title = EXCLUDED.title,
            status = EXCLUDED.status,
            assigned_to = EXCLUDED.assigned_to,
            updated_at = EXCLUDED.updated_at
            -- DO NOT overwrite created_at, commit_hash, etc. unless explicitly provided
    """, (
        task['id'], task['title'], task['status'], task['assigned_to'], time.time(),
        task.get('created_at'),  # COALESCE keeps existing if new is None
        task.get('commit_hash'),
        # ...
    ))

Key points:

ON CONFLICT DO UPDATE updates only specified columns.

Use COALESCE(EXCLUDED.col, tasks.col) to keep existing values when new data lacks a field.

Always include all columns in the INSERT part to avoid unintentional defaults.
