Bug Analysis: [DEBRIEF-BUG] Git Hooks & PYTHONPATH Issues in Worktrees
Summary
This bug report identifies two related infrastructure issues in the VETKA multi-agent video NLE system that affect worktree-based development workflows:
Git hooks (particularly post-merge) don't fire in worktrees because hooks are shared from the main .git/hooks/ directory and core.hooksPath isn't propagated to worktrees 
generate_claude_md.py --all silently fails when PYTHONPATH isn't set correctly or the virtual environment is broken
Root Cause Analysis
Issue 1: Git Hooks in Worktrees
According to the git documentation and community reports, git worktrees share the same .git directory structure but have private subdirectories in $GIT_DIR/worktrees/<id>/ . While hooks are technically shared from the main repository, the core.hooksPath configuration is not automatically propagated to worktrees .
In the VETKA codebase, I can see the project uses a .claude/worktrees/ structure (referenced in experience_report.py and role_memory_writer.py). When agents operate from worktrees:
The post-merge hook (which likely triggers generate_claude_md.py --all) won't execute
Experience reports may not be properly synchronized
CLAUDE.md regeneration may be skipped after merges
Issue 2: PYTHONPATH & Silent Failures
The generate_claude_md.py script relies on proper PYTHONPATH configuration to import from src/. When:
The virtual environment is broken or not activated
PYTHONPATH isn't set correctly
The script is called from a worktree context with different path resolution
The script may fail silently (exit 0 but produce no output or partial output), leading to stale CLAUDE.md files .

import sys
import subprocess
from pathlib import Path

def resolve_project_root() -> Path:
    """Same logic as experience_report.py for consistency."""
    candidate = Path(__file__).resolve().parent.parent
    parts = candidate.parts
    try:
        wt_idx = parts.index(".claude")
        main_root = Path(*parts[:wt_idx])

            return main_root
    except ValueError:

def ensure_pythonpath():
    """Ensure src/ is in PYTHONPATH."""
    project_root = resolve_project_root()
    src_path = str(project_root / "src")
    
    current_pypath = os.environ.get("PYTHONPATH", "")
    if src_path not in current_pypath:
        os.environ["PYTHONPATH"] = f"{src_path}{os.pathsep}{current_pypath}" if current_pypath else src_path

def validate_venv() -> bool:
    """Check if virtual environment is functional."""
    try:
        result = subprocess.run(
            [sys.executable, "-c", "import sys; print(sys.prefix)"],
            capture_output=True, text=True, timeout=5
        )
        return result.returncode == 0
    except Exception:
        return False

def main():
    # Validate environment

        print("ERROR: Virtual environment appears broken", file=sys.stderr)
        sys.exit(1)

    if claude_md.stat().st_size < 100:
        print("ERROR: CLAUDE.md appears to be empty/invalid", file=sys.stderr)
        sys.exit(1)
    
    print(f"Successfully regenerated {claude_md}")
    return 0

if __name__ == "__main__":
