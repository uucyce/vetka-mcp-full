def _resolve_project_root() -> Path:
    """Return main repo root, bypassing git worktree indirection."""
    candidate = Path(__file__).resolve().parent.parent.parent
    parts = candidate.parts
    try:
        wt_idx = parts.index(".claude")
        # Found .claude — go up to the git repo root (parent of .claude)
        main_root = Path(*parts[:wt_idx])
        if main_root.exists():
            return main_root
    except ValueError:
        pass
    return candidate

_PROJECT_ROOT = _resolve_project_root()

Logic:

Walk up from __file__ to candidate root

If path contains .claude (worktree marker), strip everything up to that point

Return parent of .claude (true repo root)

Fallback to original candidate for non-worktree runs

Additionally, role_memory_writer.py now stores reports in ~/.claude/projects/.../memory/roles/ — a shared location outside the repo that all worktrees can write to.

Lessons Learned / Recommendations

Never assume Path(__file__).resolve() yields the repo root when git worktrees are in use. Always check for .claude/ in the path.

Two-tier testing is non-negotiable for file-system-dependent code:

Tier 1 (source-parsing): Fast, cheap, catches logic/type errors

Tier 2 (live API): Slower but catches environmental mismatches

Store shared state (experience reports, role memories) outside worktree directories — either in ~/.claude/ or in the main repo root after stripping worktree components.

Add a test fixture that runs in an actual git worktree to prevent regression.
