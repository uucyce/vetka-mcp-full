DEBRIEF: API/Worker Preset & Method Alignment Bug

Agent: Beta (cross-cutting)
Task: VETKA video NLE (React + FastAPI + FFmpeg)
Phase: Post-203
Severity: Medium (blocked dispatch flows, required manual bridge)

Q1: What was the issue?

Two related inconsistencies were discovered between the Routes layer (Sonnet – FastAPI endpoints) and the Worker layer (FFmpeg execution):

Preset key mismatch

Routes defined its own PLATFORM_PRESETS dictionary (keys: "youtube", "twitter", "instagram")

Worker used CODEC_PRESETS (keys: "yt", "tw", "ig")

No direct mapping → jobs failed when a route passed platform="youtube" but the worker expected "yt".

A manual _PRESET_MAP bridge had to be added to translate keys.

Method naming divergence

Routes exposed endpoints like /cancel_job/{job_id} and /get_job_status/{job_id}

Worker internal methods were named cancel(job_id) and get_status(job_id)

Cross‑file calls (e.g., from task dispatch) broke due to name mismatch.

Root cause: Independent development of route handlers and worker core without a shared interface contract. Sonnet (routes) and the worker team used different naming conventions and data schemas.

Impact:

Manual patch (_PRESET_MAP) increased maintenance burden.

Dispatch pipeline required conditional logic to call the correct method name.

Risk of future drift as new platforms/methods are added.

Q3: What was learned? (Lessons for future)

Shared contract first – Define presets and method signatures in a common module before implementing routes or workers. Use pydantic models for API/worker boundaries.

Naming consistency – Establish a project‑wide naming style (e.g., snake_case for methods, cancel_job not cancel). Enforce with linting (e.g., ruff rule N802).

Cross‑team alignment – Run a short “interface sync” before starting parallel work on two sides of the same feature.

Test coverage – Add integration tests that send a real platform name through routes → worker and verify preset resolution. Currently missing.

Markers for drift detection – Use MARKER_XXX comments to tag shared constants, and a CI script that warns if two files define the same constant differently.
