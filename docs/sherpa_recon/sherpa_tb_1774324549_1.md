## Research Response

Research Task: [DEBRIEF-BUG] Seek-while-playing triggers excessive stopAll+playAt
🔍 Root Cause Analysis

The video playback loop (likely using requestAnimationFrame for progress updates) calls stopAll() + playAt() on every frame while playing. This is because the seek handler does not distinguish between:

User-initiated seeks (scrubbing via slider, clicking timeline, keyboard arrows)

Playback-driven time updates (automatic currentTime changes from rAF while video is playing)

Each rAF tick (typically 60 Hz) triggers a full stream restart in the FFmpeg backend, causing:

Audio clicks/pops due to decoder reinitialization

High CPU/network overhead

Jittery playback and inaccurate seeking

✅ Proposed Solution

Principle: Only re-sync the stream on user-initiated seek events, not on passive time updates during normal playback.

Implementation Strategy

Add a flag isUserSeeking (or lastUserSeekTime) to the playback controller.

Modify the seek handler to conditionally call stopAll+playAt only when the seek originates from a user action.

Debounce rapid user seeks (e.g., during scrubbing) to avoid flooding the backend with requests.

Preserve play/pause state after seeking.

  // Playback loop: only update UI time, never call seek API

    let frameId: number;
    const updateProgress = () => {
      if (!isUserSeeking && isPlaying) {
        // Fetch current stream time from backend (lightweight)
        fetchCurrentTime().then(time => setCurrentTime(time));
      }
      frameId = requestAnimationFrame(updateProgress);
    };
    if (isPlaying) frameId = requestAnimationFrame(updateProgress);
    return () => cancelAnimationFrame(frameId);
  }, [isPlaying, isUserSeeking]);

  // User-initiated seek (slider drag start)
  const handleSeekStart = () => {
    setIsUserSeeking(true);
    if (seekTimeoutRef.current) clearTimeout(seekTimeoutRef.current);
  };

  // User releases slider – perform the actual seek
  const handleSeekEnd = async (newTime: number) => {
    setIsUserSeeking(false);
    await performSeek(newTime, { resumePlayback: isPlaying });
  };

  // Debounced seek during scrubbing (optional, for smoothness)

    seekTimeoutRef.current = setTimeout(() => {
      performSeek(newTime, { resumePlayback: false }); // pause while scrubbing
