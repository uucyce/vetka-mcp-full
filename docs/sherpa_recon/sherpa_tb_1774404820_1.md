Analysis: [DEBRIEF-BUG] Lazy Imports Inside Hook Function
Root Cause
The bug occurs because lazy imports (imports inside functions) cannot be properly patched by unittest.mock.patch . When imports are declared at function scope rather than module scope, they create a local binding that bypasses the module namespace that patch targets.
Looking at the code in src/tools/auto_experience_save.py:
Python
Copy
def _get_role(branch: str):
    """Get AgentRole from registry, returns None if not found."""
    try:
        from src.services.agent_registry import get_agent_registry  # ← LAZY IMPORT
        registry = get_agent_registry()
        return registry.get_by_branch(branch)
    except Exception:
        return None

def _get_session_state() -> dict:
    """Read session tracker state."""
    try:
        from src.services.session_tracker import get_session_tracker  # ← LAZY IMPORT
        tracker = get_session_tracker()
        # ...
These function-local imports prevent clean test patching because:
patch("src.services.agent_registry.get_agent_registry") targets the module namespace
But when _get_role() runs, it performs a fresh import binding locally
The patch never intercepts this local binding 
The Fix (As Described)
The solution is to move these imports to module level:
Python
Copy
# At top of auto_experience_save.py
from src.services.agent_registry import get_agent_registry
from src.services.session_tracker import get_session_tracker

def _get_role(branch: str):
    """Get AgentRole from registry, returns None if not found."""
    try:
        registry = get_agent_registry()  # Uses module-level import
        return registry.get_by_branch(branch)
    except Exception:
        return None
Now patch("src.tools.auto_experience_save.get_agent_registry") works correctly because it patches the module namespace where the name is actually resolved .
