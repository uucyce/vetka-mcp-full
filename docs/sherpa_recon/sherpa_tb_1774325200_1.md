Research Analysis: pulseScores scene_id ↔ clip_id mismatch
🐞 Bug Summary

Location: Unknown (likely in pulse/experience tracking module, not in provided snippets)

Problem: pulseScores is a dictionary keyed by scene_id (e.g., {"scene_123": 0.85, ...}), but individual clip objects in the timeline carry a clip_id (unique per clip) with no direct reference to their parent scene.
When the system tries to assign a pulse score to a clip, it fails because clip_id is not found in pulseScores.

Impact: Pulse scores are ignored, clips don’t receive experience/affect values, breaking any downstream adaptive editing or recommendation logic.

🔍 Root Cause (Inferred)

The timeline model and pulse model are designed around different identifiers:

Timeline clips → clip_id (fine-grained, may change on split/trim)

Pulse scores → scene_id (coarser, stable across clip edits within the same scene)

There is no mapping bridge that tells a clip which scene it belongs to. Without that, pulseScores.get(clip.scene_id) is impossible because clip.scene_id doesn’t exist.

✅ Required Fix – pulse_timeline_bridge

Create a service/module that:

Parses the timeline structure (from FastAPI backend or React frontend state) to build a clip_id → scene_id map.

Enriches each clip object with a scene_id field before pulse scores are applied.

Optionally pre‑computes a scene_id → list[clip_id] index for batch operations.

# src/pulse/pulse_timeline_bridge.py

def enrich_clips_with_scene_id(timeline_data: dict) -> dict:
    """
    Given a timeline dict with scenes and clips, add `scene_id` to each clip.
    Returns the mutated timeline (or a deep copy).
    """
    for scene in timeline_data.get("scenes", []):
        scene_id = scene.get("scene_id")
        if not scene_id:
            continue
        for clip in scene.get("clips", []):
            clip["scene_id"] = scene_id   # <-- bridge field
    return timeline_data

def get_pulse_score_for_clip(clip: dict, pulse_scores: dict) -> float:
    """
    Retrieve pulse score using the enriched scene_id.
    """
    scene_id = clip.get("scene_id")
    if not scene_id:
        return 0.0   # fallback: neutral score
    return pulse_scores.get(scene_id, 0.0)
