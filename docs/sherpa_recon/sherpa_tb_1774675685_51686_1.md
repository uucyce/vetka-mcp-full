The QA verdict identifies that the "special-clean" variant is currently non-functional—it produces duplicates instead of distinct plates, and the aliasing bug remains unresolved. The root cause is likely the absence of a dedicated extraction pipeline for the special-clean role in plateLayout.ts.

2. Approach (Step-by-Step)

Step 1: Strengthen the PlateExportAssetsContract type
The current interface needs explicit fields to support "clean" variants. We must enforce that a special-clean plate requires a distinct cleanUrl and targetPlate.

Step 2: Implement Clean Plate Generation Logic
The current buildPlateExportAssetsContractModel likely treats special-clean as a fallback or alias for the background. We need to inject logic that maps a special-clean plate to its target (e.g., plate_05 generates plate_05_clean) and ensures the RGBA output is the result of removing the target object (hole-fill) rather than just masking it.

Step 3: Integrate with Depth/Hole-fill Pipeline
Leverage the holeFill metadata from the canonical layer_space.json. A special-clean plate should point to the holefill_rgba.png asset generated by the extraction script, not the standard mask.

Step 4: Fix Aliasing in E2E Export
The "aliasing bug" suggests the export is pulling the wrong image data. We need to ensure the exportProxyAssets function in App.tsx resolves the correct blob URL for the clean variant.

  sample: SampleMeta,

  const processedPlates = plates.map(plate => {
    // Handle special-clean role with distinct asset generation
    if (plate.role === 'special-clean' && plate.targetPlate) {
      const targetPlate = plates.find(p => p.id === plate.targetPlate);
      
      if (targetPlate) {
        // Generate distinct clean plate filename
        // e.g., plate_05 -> plate_05_clean_rgba.png
        const cleanRgbaName = `${plate.targetPlate}_clean_rgba.png`;
        const cleanMaskName = `${plate.targetPlate}_clean_mask.png`;
        const cleanDepthName = `${plate.targetPlate}_clean_depth.png`;
        
        return {
          ...plate,
          rgbaUrl: cleanRgbaName,
          maskUrl: cleanMaskName,
          depthUrl: cleanDepthName,
          cleanUrl: cleanRgbaName,
          cleanConfig: {
            isCleanVariant: true,
            sourceTargetId: plate.targetPlate,
            holeFillMethod: 'lama',
            holeFillAssetPath: `layer_${plate.targetPlate}_holefill_rgba.png`
          }
        };
      }
    }
    return plate;
  });

  return {
    contract_version: "1.0.0",
    sampleId: sample.id,
    sourceUrl: sample.sourceUrl,
    globalDepthUrl: `${sample.id}_depth.png`,
    backgroundRgbaUrl: `${sample.id}_background_rgba.png`,
    backgroundMaskUrl: `${sample.id}_background_mask.png`,
