## Research Response

Analysis

Bug confirmed: The render engine already supports set_clip_color for applying color grading, but the UI never calls this operation when the user adjusts color grading. Additionally, the persistence layer does not store the color grading values, so they are lost on project reload.

Root cause gap:

React UI missing API call to backend on color grading change.

Backend missing handler to persist color grading to project file and forward to render engine.

Proposed Solution
1. Frontend (React) – Add API call on color grading change

In the color grading component (e.g., ColorGradingPanel.tsx), add a debounced PATCH request when sliders/LUT change:

src/routes/clips.py (or similar):

from src.services.render_engine import RenderEngine
from src.services.project_persistence import ProjectStore

router = APIRouter()

@router.patch("/clips/{clip_id}/color_grading")
async def update_clip_color_grading(
    clip_id: str,
    grading: ColorGradingSchema,
    project_id: str = Query(...),
    db: Session = Depends(get_db),
):
    # 1. Update in-memory timeline state
    clip = timeline.get_clip(clip_id)
    clip.color_grading = grading.dict()
    
    # 2. Persist to project file (or DB)
    project = ProjectStore.get(project_id)
    project.update_clip(clip_id, {"color_grading": grading.dict()})
    await ProjectStore.save(project)          # writes to disk/db
    
    # 3. Tell render engine to apply new grading
    render_engine = RenderEngine.get_instance()
    render_engine.set_clip_color(clip_id, grading.to_ffmpeg_params())
    
    return {"status": "ok"}

Persistence must save the color_grading field inside the clip’s JSON in the project file (.vetka or database). Ensure the project loader reads it back.
