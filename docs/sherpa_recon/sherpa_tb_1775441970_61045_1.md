Fix: Reorder startImport before openFilePicker in ProjectPanel.tsx
Problem

In client/src/components/cut/ProjectPanel.tsx:

Line 333: openFilePicker useCallback includes startImport in its dependency array.

Line 384: startImport is declared with const startImport = useCallback(...).

On first render, startImport is undefined inside the closure of openFilePicker. The first Cmd+I press fails because the import function is stale.

Solution
