2. Chosen Scenario: B – Worktree-local memory

Decision: Each agent stores its memory in .claude/worktrees/{worktree_name}/memory/ (where {worktree_name} matches the role’s callsign, e.g., Alpha, Beta). Watchdog is configured to scan all worktree memory directories.

Justification:

Isolation by design – Worktrees already separate agent code, CLAUDE.md, and task context. Memory belongs to the same execution boundary.

No cross-role leakage – RAG vectors from Alpha’s memory never mix with Beta’s because physical directories differ.

Simplifies agent instructions – Each agent writes to ./memory/ relative to its worktree root (already cwd when running via claude_code).

Aligns with existing patterns – generate_claude_md.py writes per-worktree CLAUDE.md; owned_paths in agent_registry.yaml already includes .claude/worktrees/*/CLAUDE.md. Extending to memory/ is natural.
