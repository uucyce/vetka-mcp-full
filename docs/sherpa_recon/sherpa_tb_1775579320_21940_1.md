Research & Implementation Plan: Canonical Motion Presets for Headless Parallax Package
1. Overview

Goal
Add a set of headless motion presets (push-in, push-out, pan, orbit‑like drift, subtle handheld) with independent speed and intensity controls, directly usable without a timeline UI. The motion must be compatible with the existing packaged parallax scene (multi‑plate + depth maps + clean plates) and must work in a headless pipeline (CLI / script / importable package).

Current state (from docs)

Multi‑plate renderer exists (photo_parallax_render_preview_multiplate.py) and can use per‑plate depth maps.

No explicit camera geometry yet – motion is driven by heuristics (parallaxStrength, travelXPct).

AE reference confirms pinhole camera + zoom/focal length model is the desired final path.

This task
Does not require a full camera geometry refactor. Instead, we implement motion presets as transform generators that output per‑frame 2D transforms (translate, scale, rotate) for each plate, respecting relative depth ordering. This keeps the change isolated, works with the current renderer, and is immediately useful for Pavel. Speed/intensity controls map directly to preset parameters (e.g., speed = 1.0 → full travel in 5 seconds, intensity = 0.5 → half the max displacement).

2. Design Decisions
2.1 Motion Representation

For headless use and AE compatibility, the simplest canonical form is a CSV / JSON keyframe list per plate:
time (0..1), translate_x (0..1), translate_y (0..1), scale (factor), rotate (deg).

The renderer then samples these keyframes at each output frame. This is renderer‑agnostic, easily serialisable, and can be imported into After Effects via a script.

2.2 Depth‑Awareness

Presets must respect the scene’s depth ordering:

Foreground plates move more (larger parallax) than background plates.

We achieve this by defining a global camera path (e.g., dolly forward, truck left) and projecting it onto each plate’s Z‑depth.

For simplicity in this iteration, we pre‑compute a per‑plate parallax factor = 1 / (depth_normalized + epsilon), then scale the camera motion by that factor.

If depth maps are not yet per‑plate, we fall back to a reasonable default ordering (layer index).

2.3 Preset Parameters

Each preset exposes:

Parameter	Type	Range	Effect
speed	float	0.2 – 2.0	Scales the total duration of the motion (relative to a 5s reference).
intensity	float	0.0 – 1.0	Scales the amplitude (max displacement, max scale change, max rotation).

Additionally, some presets have unique parameters (e.g., direction for pan, handheld_frequency for subtle drift).

2.4 Headless Entry Point

We will add a CLI script generate_motion.py that:

Takes an input scene folder (plates + depth + metadata)

Applies a chosen preset with given speed/intensity

Exports either:

A rendered video (using existing renderer + generated transforms)

An AE‑import package (transforms + plates + depth)

def compute_plate_transforms(camera_path_func, num_frames, fps, plates_metadata, intensity, speed):
    # plates_metadata: list of dicts with 'depth_normalized' (0=bg, 1=fg)
    # speed: adjust total duration (e.g., 2.0 = half duration)
    total_duration = 5.0 / speed  # 5 seconds reference duration
    times = np.linspace(0, 1, num_frames)
    transforms = []
    for t in times:
        cam_zoom, cam_dx, cam_dy, cam_rot = camera_path_func(t, intensity)
        frame_transforms = {}
        for plate in plates_metadata:
            parallax_factor = 1.0 / (plate['depth_normalized'] + 0.1)
            # apply parallax scaling
            dx = cam_dx * parallax_factor * intensity
            dy = cam_dy * parallax_factor * intensity
            zoom = 1.0 + (cam_zoom - 1.0) * parallax_factor
            rot = cam_rot * parallax_factor
            frame_transforms[plate['id']] = (dx, dy, zoom, rot)
        transforms.append(frame_transforms)
    return transforms  # list of dicts per frame
