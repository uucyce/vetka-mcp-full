Research Summary: F9/F10 Aliases Duplicate Handler Logic
Problem Analysis
The bug identifies a classic React architecture issue where F9 and F10 hotkey aliases duplicate insertEdit/overwriteEdit handler logic inline because useMemo closures cannot self-reference the same handler function. This is a code duplication anti-pattern that violates DRY principles .
Root Cause
In React's useMemo hook, closures cannot easily reference themselves or share memoized callbacks across multiple key bindings. When you have:
F9 → insertEdit action
F10 → overwriteEdit action
And potentially other aliases for the same actions
The current implementation likely inlines the same handler logic for each key, creating maintenance overhead and bug risks.
Recommended Architecture Fix: Multi-Binding Per Action
Based on established React keybinding patterns , the architecture should support multiple keys bound to a single action handler via string[] in the HotkeyMap configuration.

  F10: () => { /* overwriteEdit logic */ }, // different action

// HotkeyMap with string[] support for multiple keys per action
interface HotkeyAction {
  keys: string[];           // Multiple bindings: ["F9", "NumpadInsert"]
  handler: () => void;      // Single handler reference
  description: string;
  context?: string;
}

const HOTKEY_MAP: Record<string, HotkeyAction> = {
  insertEdit: {
    keys: ["F9", "NumpadInsert"],  // Multiple aliases supported
    handler: insertEditHandler,      // Single reference, no duplication
    description: "Insert edit at playhead",
    context: "timeline"
  },
  overwriteEdit: {
    keys: ["F10", "NumpadDecimal"],
    handler: overwriteEditHandler,
    description: "Overwrite edit at playhead",
    context: "timeline"
  }
};

// Hook implementation with multi-binding support
function useMultiKeybindings(actions: Record<string, HotkeyAction>) {
  useEffect(() => {
    const keyToAction = new Map<string, () => void>();
    
    // Flatten multi-key bindings into lookup map
    Object.values(actions).forEach(action => {
      action.keys.forEach(key => {
        keyToAction.set(key.toLowerCase(), action.handler);
      });
    });

    const handleKeyDown = (e: KeyboardEvent) => {
      const handler = keyToAction.get(e.key.toLowerCase());
      if (handler) {
        e.preventDefault();
        handler();
      }
    };

    window.addEventListener("keydown", handleKeyDown);
    return () => window.removeEventListener("keydown", handleKeyDown);
  }, [actions]);
}
