Critical Issue: LiteLLM Parsing Gap
According to GitHub issue #24091 , LiteLLM fails to parse Ollama's tool_calls even when Ollama returns them correctly. The issue states:
"When using LiteLLM to call an Ollama model with tool definitions, the response from Ollama contains a valid tool_calls field, but LiteLLM does not expose it in the returned ModelResponse object."
This means even if Gemma4 outputs native tool_calls, LiteLLM may not forward them properly to free-code.
Gemma4's Native Tool Format
Gemma4 uses special tokens for tool calling  :
<|tool_call> / <tool_call|> delimiters
<|"|> for string escaping
Example: <|tool_call>call:get_current_time{timezone:<|"|>Asia/Tokyo<|"|>}<tool_call|>
Ollama 0.20.2+ has built-in parsing for this format , converting it to standard OpenAI tool_calls format.
