## Overview

Persistent role memory gives every VETKA agent a `MEMORY.md` file that auto‑populates from `Q1/Q2/Q3` debrief data. This closes the broken feedback loop: what an agent learns today is injected into its next session, enabling experiential learning across sessions.

The design follows the proven Sherpa JSONL pattern (auto‑collect → structured store → inject into next decision). Burnell’s manual `MEMORY.md` serves as the proof of concept.
