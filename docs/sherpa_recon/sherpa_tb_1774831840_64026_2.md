2. Approach – Step‑by‑Step Plan

Audit existing left‑rail content
Identify all distinct UI sections:

Object / layer cards (likely dynamic list from groupBoxes or plates)

Scene Plan settings (stage depth, focus, motion)

Effect settings (matte seeds, brush hints, AI assist)

Debug / advanced toggles

Define “operator‑critical” items

Must stay (concise):

Layer visibility toggle, layer selection, basic transform (position/scale)

Focus rectangle (x,y,width,height) with one numeric row

Motion travel (X%, Y%, zoom) as a compact slider group

Render / preview mode selector (button strip)

Can be collapsed into a summary accordion:

Scene Plan (depth layers, phase) → show only “Layers: 5” and a “Configure” link

Group boxes → replace list with a count badge + “Edit” button

Matte seeds → show seed count + “Manage”

Remove completely (low signal):

Redundant object cards that only repeat layer name

Any “non‑functional extract card” (referenced in earlier Sherpa tasks)

Duplicate settings that also appear in a right‑side inspector

Redesign left rail as a DaVinci‑like “Inspector”

Fixed header with project name / layer count (no scroll until content overflows)

Compact sections (each collapsible, with a summary line when closed)

Scene → Focus + Motion (two rows)

Layers → Compact table (name, visibility, lock, Z‑order controls)

Matte → Seed count, brush size, mode selector (single row)

AI Assist → Single button row (“Suggest” / “Apply”)

No nested scroll inside sections – instead rely on a single scrollbar for the whole left rail, with max‑height limited by viewport.

Implement CSS changes

Reduce left‑rail width from e.g. 320px → 280px (DaVinci uses ~260‑280px)

Use gap: 0.5rem, padding: 0.75rem, font-size: 0.85rem

Replace list‑style object cards with a <div> grid or flex‑wrap of layer pills

Add overflow-y: auto and max-height: 100vh to prevent “endless” feeling

Add a “Compact Mode” toggle (optional)

For power users, a toggle that further collapses non‑essential sections into icon‑only rows, mimicking DaVinci’s “Hide details” mode.

4. Risks & Edge Cases
Risk	Mitigation
Loss of functionality – Removing “endless” cards might hide settings that some users rely on (e.g., per‑layer Z‑order fine‑tuning).	Move detailed per‑layer controls into a right‑click context menu or a modal “Layer Details” panel. Keep Z‑order arrows in the compact layer row.
Screen real‑time feedback – Collapsing sections may hide active matte seeds, making manual hint placement harder.	Show a small live preview (e.g., dot grid) inside the summary line, or keep the matte seed count badge and a “Show Overlay” toggle.
DaVinci reference misinterpretation – DaVinci has multiple workspaces (Cut, Edit, Fusion). Our “operator” layout should mimic the Edit page inspector (compact, tabbed).	Provide a reference screenshot of DaVinci’s left inspector (clip attributes) and annotate which elements map to our controls.
Existing state dependencies – Many settings (group boxes, hints) are stored in window.vetkaParallaxLab; removing UI elements must not break the API.	Keep all internal state and API methods intact; only change the visual representation. Use display: none or conditional rendering, never delete state variables.
Accessibility – Compact mode might reduce touch target sizes.	Ensure buttons are at least 32×32px, use aria-label for icon‑only controls.
