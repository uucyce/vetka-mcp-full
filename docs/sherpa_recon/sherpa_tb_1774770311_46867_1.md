Summary (what to change to stop overlap)

The “giant text + overlapping labels” symptom in workflow cards is almost always a combo of:

typography that scales too aggressively (often vw-based without sane caps), and
flex/grid children that refuse to shrink (default min-width:auto / “min-content” behavior), so labels/chips collide instead of wrapping/truncating. Setting min-width: 0 on the right flex/grid children is the standard fix. 
1

3.2 App.tsx — small structural wrappers so CSS can do its job

In App.tsx, you want two things:

A .workflow wrapper around the workflow area (so typography changes don’t accidentally affect the stage viewport / whole app), and
Consistent header markup for each “card” so badges/labels can wrap.

A minimal pattern (drop-in helper component inside App.tsx):

React
function WorkflowCard(props: {
  title: string;
  badges?: Array<string>;
  children: React.ReactNode;
}) {
  const { title, badges, children } = props;

  return (
    <section className="workflowCard">
      <header className="workflowCardHeader">
        <h3 className="workflowCardTitle">{title}</h3>

        {badges?.length ? (
          <div className="workflowCardBadges">
            {badges.map((b) => (
              <span key={b} className="workflowBadge" title={b}>
                {b}
              </span>
            ))}
          </div>
        ) : null}
      </header>

      <div className="workflowCardBody">{children}</div>
    </section>
  );
}

Then, wherever you currently render workflow cards (Import/Depth/Extract/Camera/Export/Inspector/Routing…), wrap the area:

React
<div className="workflow">

  <WorkflowCard title="Depth" badges={["white = near", "black = far"]}>

4) Risks (edge cases / what can still break)

Unbroken strings (hashes, long filenames, model IDs)

Without overflow-wrap:anywhere, a single long token can still force overflow and trigger collisions. This patch explicitly addresses that. 
4

Nested flex containers

If your header has nested flex children, you may need min-width: 0 at multiple levels (parent row + the specific child that contains text). This is a known flexbox gotcha behind many “why is text overlapping?” bugs. 
1

Chip readability vs truncation

Ellipsis prevents overlap but can hide meaning. Using title="" helps, but if operators rely on seeing full routing reasons at-a-glance, you may prefer wrapping chips (remove white-space: nowrap) in a second iteration.

Global typography side-effects

If you apply tokens to body instead of .workflow, you might unintentionally shrink stage labels/debug overlays. Scoping to .workflow avoids that.
