Debrief: Worktree Path Resolution Bug (Fixed)
Summary

Three Sonnet agents (Alpha, Beta, Gamma) working in parallel identified a critical path resolution bug via static source analysis in ~2 minutes. The bug caused experience reports, task board data, and role memories to be written to worktree-local directories (.claude/worktrees/<name>/data/) instead of the main repository data/ directory. This broke downstream processes (CLAUDE.md generator, session init, heartbeat) that expected reports in the main repo.

Root cause: Path(__file__).resolve().parent.parent.parent returns the worktree root when running inside a git worktree (e.g., .claude/worktrees/harness/src/...). No detection logic existed to climb up to the actual main repo root.

Manifestation: Experience reports “disappeared” — saved but never found by the CLAUDE.md generator. No error logs because the directory existed (worktree has its own data/).

Fix Applied (4 locations)
File	Function	Fix
src/services/experience_report.py	_resolve_project_root()	Walk path parts, detect .claude/worktrees/, return parent of .claude
src/orchestration/task_board.py	_resolve_main_repo_root()	Same pattern + fallback to VETKA_MAIN_REPO env var
src/memory/role_memory_writer.py	_resolve_project_root() + _get_roles_memory_dir()	Same root resolution; write to ~/.claude/projects/.../memory/roles/ (worktree‑safe by design)
src/tools/auto_experience_save.py	(truncated, but same pattern applied)	Uses shared root resolution

Verification: The fix was validated by running python -c "from src.services.experience_report import _PROJECT_ROOT; print(_PROJECT_ROOT)" from inside a worktree – correctly printed main repo path.
