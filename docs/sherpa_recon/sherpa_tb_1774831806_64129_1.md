PARALLAX-RECON: Extract card, left inspector rail & cleanup sheet audit
Executive summary

Current UI in photo_parallax_playground suffers from:

Extract block – present as a second‑level card but lacks actionable settings; appears read‑only or decorative.

Left rail – hosts 8+ panels (Focus Proxy, Guided Hints, Stage Tools, Algorithmic Matte, Hint Brushes, Merge Groups, AI Assist, debug toggles) → creates “endless settings strip” and recenters content unnecessarily.

Cleanup sheet – split across multiple modes (brush, matte seeds, group boxes) that are not context‑aware, generating visual noise.

Model‑centric vs operator‑centric – confirmed by PARALLAX_UI_OPERATOR_ROADMAP_2026-03-29.md: too many tools visible simultaneously, weak contextual binding.

Key finding: Extract is read‑only (no user‑editable parameters) and serves only as a static label or placeholder. Left rail contains debug panels (Debug Snapshot), redundant controls (Focus Proxy with stage already visible), and over‑exposed advanced tools (Algorithmic Matte, AI Assist). Cleanup sheet lacks a single “rescue tool” modal – instead spreads brushes, seeds, and groups as independent sections.
