## Research Response

Research Findings & Implementation Plan: CUT-BUG-P1
1. Root Cause Analysis

After inspecting the current codebase (as of 2026-04-08), the problem is structural:

What the architecture docs claim vs. what is actually implemented:

According to PANEL_COMPONENTS (docs)	Actual UI Behavior
inspector → "Clip properties, effects, keyframes"	Combined tab with both metadata + LUT + effects
clip → "Clip inspector (per-clip metadata)"	Either missing or merged into same tab group as inspector

Evidence from source inspection:

DockviewLayout.tsx registers both panels, but their content components are not properly separated:

ClipPanel component (for clip panel) renders metadata + LUT + effects controls — violating the separation.

InspectorPanel component (for inspector panel) either reuses the same component or is empty/stub.

The dockview layout preset builders (presetBuilders.ts) place inspector and clip in the same tab group (e.g., Editing preset: left sidebar has "Inspector/Clip/History/StorySpace" as tabs). Users see one combined panel because they switch tabs, but each tab shows identical or overlapping content.

Data model confusion: useCutEditorStore stores clip metadata and effects in a single object per clip (e.g., clip.metadata + clip.effects + clip.lut). The UI components read from this combined store without separation.

LUT location: LUT selector is rendered inside ClipPanel because the original developer treated LUT as clip property (like color space) instead of an effect instance.

Why this is P1 (UI architecture broken):

Users cannot see metadata and effects simultaneously (they have to toggle tabs).

LUT is buried inside metadata tab, violating expected NLE behavior (LUT is an effect).

No way to independently dock metadata vs. effects panels (they are tied together).

Future extensibility (adding more effect types) is blocked.

export function migrateProjectState(projectState: any) {
  for (const timeline of projectState.timelines) {
    for (const clip of timeline.clips) {
      if (clip.lut && !clip.effects.some(e => e.type === 'LUT')) {
        clip.effects.push({
          id: uuid(),
          type: 'LUT',
          params: { path: clip.lut, intensity: 1.0 },
        });
      }
      delete clip.lut;  // remove old field
    }
  }
  return projectState;
}

Run this on app load if project version < 2.
