Bug Analysis: _resolve_project_root() Worktree Detection Heuristic
The Problem
The current implementation in experience_report.py and role_memory_writer.py uses a fragile path-based heuristic:

def _resolve_project_root() -> Path:
    candidate = Path(__file__).resolve().parent.parent.parent
    parts = candidate.parts
    try:
        wt_idx = parts.index(".claude")
        main_root = Path(*parts[:wt_idx])
        if main_root.exists():

Issues with this approach:
Hardcoded .claude directory name — If the worktree structure changes (e.g., worktrees moved to a different location, or the .claude directory is renamed), the detection fails silently.
No verification that found path is actually a git repo — The code only checks main_root.exists(), not whether it contains a .git directory.
Inconsistent with task_board.py — The task board uses a different approach (VETKA_MAIN_REPO env var → git rev-parse), creating maintenance divergence.
The Correct Solution
Per Git documentation and best practices , git rev-parse --show-toplevel is the authoritative way to determine the repository root. It correctly handles:
Normal repositories
Worktrees (returns the worktree root, not the bare repo) 
Subdirectory execution (works from any depth)

Replace the .claude string-matching heuristic with git rev-parse --show-toplevel. Here's the unified pattern all three files should use:

import subprocess
from pathlib import Path

def _resolve_project_root() -> Path:
    """Return main repo root using git rev-parse --show-toplevel.
    
    This is the authoritative method that works correctly with:
    - Normal git repositories
    - Git worktrees (linked or bare-repo based)
    - Execution from any subdirectory depth
    """
    # Priority 1: Environment override (for containerized/CI environments)
    env_root = os.environ.get("VETKA_MAIN_REPO")
    if env_root and Path(env_root).is_dir():
        return Path(env_root)
    
    # Priority 2: Git's authoritative answer
    try:
        result = subprocess.run(
            ["git", "rev-parse", "--show-toplevel"],
            capture_output=True,
            text=True,
            check=True,
            timeout=5,
        )
        git_root = result.stdout.strip()
        if git_root:
            path = Path(git_root)
            if path.is_dir():
                return path
    except (subprocess.CalledProcessError, subprocess.TimeoutExpired, FileNotFoundError):
        pass
    
    # Priority 3: Fallback to __file__-relative (for non-git environments)
    return Path(__file__).resolve().parent.parent.parent

def _resolve_shared_data_root() -> Path:
    """Find the main repo's data directory, shared across all worktrees."""
    # Get the common git directory (main .git location)
    result = subprocess.run(
        ["git", "rev-parse", "--git-common-dir"],
        capture_output=True,
        text=True,
        check=True,
    )
    git_common = Path(result.stdout.strip())
    # The main repo root is parent of .git
    main_root = git_common.parent if git_common.name == ".git" else git_common
    return main_root / "data"
This ensures reports written from any worktree are stored in the main repository's data/ directory, accessible to all other processes.
