Platform API Analysis
1. YouTube (Google) — Recommended First Platform
OAuth 2.0 Flow:
Standard OAuth 2.0 with authorization code flow
Requires Google Cloud Console app registration
Scopes needed: https://www.googleapis.com/auth/youtube.upload, https://www.googleapis.com/auth/youtube.readonly
Upload Method:
Resumable upload protocol (similar to Vimeo's tus)
Supports video metadata (title, description, tags, thumbnails)
Quota-based rate limiting (10,000 units/day default)
Implementation Notes:
Most mature API of the four platforms
Well-documented Python client (google-api-python-client, google-auth)
Supports scheduled publishing via publishAt parameter
2. Instagram Graph API — Most Complex
Critical Constraints :
Business/Creator accounts only — Personal accounts cannot use the API
Container-based publishing — Create container → Wait for processing → Publish
Two OAuth paths — Instagram via Facebook OR Instagram Direct
Strict media requirements — Aspect ratio, resolution, bitrate validation
App review required — Detailed screen recordings and use-case justification
Upload Flow:
Create media container (POST /{ig-user-id}/media)
Upload video to container
Wait for processing (poll status)
Publish container (POST /{ig-user-id}/media_publish)
3. TikTok Content Posting API — Most Restrictive
Critical Constraints :
Gated access — Must apply for video.upload and video.publish scopes
App audit required for public posting — "API Clients must not be limited to test applications and should be intended for a wide audience"
Private-only for unaudited apps — All content restricted to private viewing until audit passed
OAuth 2.0 with quirks — Uses client_key instead of standard client_id 
Upload Flow:
Initialize upload session (/post/publish/inbox/video/init/)
Upload video bytes (chunked)
Publish with metadata (/post/publish/video/)
Warning: TikTok explicitly prohibits "utility tools to help upload contents to the account(s) you or your team manages" . VETKA may need to frame itself as a "creator platform" rather than an internal tool to pass audit.
4. Vimeo API — Simplest OAuth
Authentication :
OAuth 2.0 OR Personal Access Tokens (PAT)
PAT recommended for single-user scenarios
Scopes: upload, edit, private
Upload Methods :
Tus resumable (recommended for large files)
Form-based (simple HTML form)
Pull (Vimeo fetches from URL)
Python SDK: Official vimeo-python library handles tus protocol automatically

-- Social platform accounts (encrypted tokens)
CREATE TABLE social_accounts (
    id UUID PRIMARY KEY,
    user_id UUID REFERENCES users(id),
    platform VARCHAR(20) NOT NULL, -- 'youtube', 'instagram', 'tiktok', 'vimeo'
    account_name VARCHAR(255),
    access_token_encrypted BYTEA NOT NULL,
    refresh_token_encrypted BYTEA,
    token_expires_at TIMESTAMP,
    platform_account_id VARCHAR(255), -- YouTube channel ID, etc.
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW(),
    UNIQUE(user_id, platform, platform_account_id)
);

-- Cross-post jobs
CREATE TABLE crosspost_jobs (
    id UUID PRIMARY KEY,
    project_id UUID REFERENCES projects(id),
    render_job_id UUID REFERENCES render_jobs(id), -- Link to export
    status VARCHAR(20) NOT NULL, -- 'pending', 'uploading', 'processing', 'published', 'failed'
    created_at TIMESTAMP DEFAULT NOW(),
    completed_at TIMESTAMP
);

Security Requirements
Token Storage
Never store plaintext credentials
Use Fernet symmetric encryption or AWS KMS/GCP KMS for token encryption at rest
Encrypt with backend-only key (never expose to frontend)
OAuth Security
Use PKCE (Proof Key for Code Exchange) for all OAuth flows where supported
Store state parameter in session to prevent CSRF
Short-lived authorization codes (10 min max)
Refresh token rotation on use
