1 · Extract — Current Behavior Audit
1.1 What Extract most likely does today

In a photo-parallax pipeline the word Extract conventionally names the step that calls the depth-estimation backend (e.g. Depth-Anything-v2 / MiDaS via FastAPI) and stores the resulting depth map as a side-channel asset. It is a pipeline stage, not a transform—it has no tunable parameters for an operator once the model is chosen.

2 · Left-Rail / Inspector Noise Audit
2.1 Identified noise sources

Based on the user report ("endless ribbon of unclear controls plus object lists") and common React inspector anti-patterns, the following are the specific sources:

Noise Source A — Flat control stack (no sectioning)

The inspector likely renders all controls for all effects in one scrollable column, conditionally shown via inline display: none toggling rather than mounting only the active section. This means every effect's DOM exists simultaneously, inflating the visual height and causing scroll overshoot.

React
// Anti-pattern (current, likely):
<div className="left-inspector">
  <UploadControls />        {/* always mounted */}
  <ExtractControls />       {/* always mounted, empty */}
  <WarpControls />          {/* always mounted */}
  <AnimateControls />       {/* always mounted */}
  <ObjectList objects={objects} />   {/* unbounded list */}
</div>
Noise Source B — ObjectList inside the inspector

The object/layer list (segmented regions or parallax layers) is almost certainly rendered inside the same left rail as effect controls. This is the DaVinci antipattern that Resolve itself fixed by separating the Node Editor (object topology) from the Inspector (per-node parameters). Mixing them produces the "ribbon" effect.

Noise Source C — Re-centering on open

When the inspector mounts, it likely calls scrollIntoView() or resets scrollTop on the parent container without constraining the scroll region, causing the viewport to jump. Alternatively, a CSS height: 100vh on the inspector forces the outer layout to reflow.

CSS
/* Anti-pattern in index.css (likely): */
.left-inspector {
  height: 100vh;          /* forces full-page reflow */
  overflow-y: auto;       /* unbounded scroll */
}
Noise Source D — Missing collapse/expand state

Every sub-section (warp parameters, animation keyframes, object properties) is rendered fully expanded. 
12
A noise-reduction control window should divide controls into major areas with proper grouping—without that separation, the inspector shows all controls lined up next to each other without any grouping whatsoever. The same principle applies here.

3 · DaVinci-like Conversion Target
3.1 The DaVinci Resolve AI Depth Map panel as reference
7
 The Depth Map effect in DaVinci Resolve allows you to generate a 3D depth matte of a scene automatically and makes it possible to quickly grade the foreground separately from the background.

Its inspector panel is organized as follows (used as our template):

3
 By default, the Depth Map Preview is on, showing the black and white matte. Near Limit (white point) and Far Limit (black point) sliders define the overall depth range. The Gamma slider adjusts the midtones. 
3
 To select a specific depth area, enable Isolation. Use Target Depth (1=foreground, 0=background) to pick a focus point, Tolerance to set the range around it, and Softness for a smoother fall-off. 
3
 Depth Map can be GPU-intensive. Use the Quality dropdown (Faster for tweaking, Better for final output) for better performance.

import React, { useState } from 'react';

interface Props {
  title: string;
  defaultOpen?: boolean;
  children: React.ReactNode;
}

export const EffectSection: React.FC<Props> = ({
  title, defaultOpen = false, children
}) => {
  const [open, setOpen] = useState(defaultOpen);
  return (
    <div className="effect-section">
      <button
        className={`effect-section__header ${open ? 'open' : ''}`}
        onClick={() => setOpen(o => !o)}
        aria-expanded={open}
      >
        <span className="effect-section__title">{title}</span>
        <span className="effect-section__chevron">{open ? '▾' : '▸'}</span>
      </button>
      {open && (
        <div className="effect-section__body">
          {children}
        </div>
      )}
    </div>
  );
};

import { useState, useCallback, useRef } from 'react';

export const useDepthMapControls = () => {
  const [config, setConfig] = useState<DepthMapConfig>({
    previewEnabled: true,
    nearLimit: 255,
    farLimit: 0,
    gamma: 1.0,
    isolationEnabled: false,
    targetDepth: 0.5,
    tolerance: 0.2,
    softness: 0.1,
    invert: false,
  });

  // Prevent recentering by preserving scroll position
  const scrollPositionRef = useRef<number>(0);

  const updateConfig = useCallback((updates: Partial<DepthMapConfig>) => {
    // Save scroll position before state update
    const inspector = document.querySelector('.compact-inspector');
    if (inspector) {
      scrollPositionRef.current = inspector.scrollTop;
    }

    setConfig(prev => ({ ...prev, ...updates }));

    // Restore scroll position after render
    requestAnimationFrame(() => {
      if (inspector) {
        inspector.scrollTop = scrollPositionRef.current;
      }
    });
  }, []);

  return {
    config,
    updateConfig,
    // ... other methods
  };
};
