[DEBRIEF-BUG] Qdrant client.search() deprecated → migrate to query_points()
🔍 Analysis

Issue: Qdrant’s legacy client.search() method is deprecated and will be removed in a future version. The codebase currently uses it somewhere (not in the provided snippets – likely in src/memory/vector_store.py or src/services/qdrant_client.py).

Impact: Low priority for now, but will cause runtime errors after upgrade to Qdrant client ≥1.7.0.

Root cause: Qdrant introduced a new, more flexible query_points() API that replaces search(). The main differences:

search() (old)	query_points() (new)
collection_name: str	same
query_vector: list[float]	query: list[float] (parameter renamed)
limit: int	same (or limit inside QueryRequest)
returns list[ScoredPoint]	returns QueryResponse with .points attribute
