4. Risks & Edge Cases
Risk	Mitigation
Long unbreakable strings (e.g., base64, file paths)	Add overflow-wrap: break-word; and word-break: break-all; for code/asset paths.
Collateral styling – changes affect other parts of UI	Scope CSS to .workflow-card or use CSS modules. Avoid global * selectors.
Responsive breakpoints – text may overlap on smaller screens	Use media queries to reduce font size or switch to vertical layout at <768px.
Sliders/labels shifting – increasing padding might push controls out of view	Ensure parent containers have overflow: visible and use relative units. Test with different zoom levels (100%, 125%).
Build fails due to missing Tailwind classes	If modifying Tailwind config, run npm run build locally first. Use only existing theme keys or extend safely.
Third-party component libraries (e.g., Radix, MUI) override styles	Use higher specificity (e.g., [data-radix-wrapper] .workflow-card) or inline styles.
Portrait assets cropping – not directly related to text, but layout may shift	Ensure card containers do not have fixed aspect ratio; let them size based on content.
