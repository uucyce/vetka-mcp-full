Analysis of the bug

The bug report describes an incorrect linear shift (±0.3) applied uniformly to both 3200K (warm) and 10000K (cool) color temperatures. This is physically wrong because color temperature scaling follows the blackbody radiation curve (Planck's law) — not linear.

Why linear fails: The chromaticity coordinates of a blackbody radiator do not move linearly with temperature. A ±0.3 shift in, say, u'v' space would be wildly different in perceptual effect at 3200K vs 10000K.

Correct approach: Map temperature to CIE D‑series illuminants or use a proper Planckian locus lookup + scaling in a perceptually uniform space (e.g., CIE UCS).

import numpy as np
def temperature_to_rgb_gains(kelvin: float) -> tuple[float, float, float]:
    """
    Returns (red_gain, green_gain, blue_gain) for given temperature (K).
    Based on Planck's law and CIE 1931 2° observer.
    """
    # Normalised temperature (1000K → 0.0, 10000K → 1.0) – not linear!
    # Use a precomputed lookup table or the following approximation:
    t = kelvin / 1000.0
    if t <= 66:
        r = 1.0
        g = 0.390081578 * np.log(t) - 0.631841444
        b = 1.0 if t <= 20 else (0.543206789 * np.log(t - 10) - 1.196254356)
    else:
        r = 1.292936186 * (t ** 0.266571)
        g = 1.129890531 * (t ** 0.133071)
        b = 1.0
    # Clip and normalise (preserve luminance)
    gains = np.clip([r, g, b], 0.0, 2.0)
    return gains / max(gains)  # or return as is for multiplicative gains
