4. Risks — Edge Cases
A. Auto-open may feel too aggressive

If cleanup tools open every time the operator touches a brush, the UI may feel unstable.

Safer option
show recommendation chip first
open the lane only on explicit click
B. Discovery can get worse if everything is hidden

If the panel disappears completely, operators may think cleanup was removed.

Fix

Keep a compact summary row visible:

“No cleanup task open”
“Open cleanup tools”
recommendation chip when relevant
C. Existing control wiring may be tightly coupled to old group layout

If current JSX assumes all cleanup controls render at once, moving them into task cards may expose:

stale state assumptions
hidden dependencies between groups
null-state rendering errors
Mitigation

Refactor in two passes:

wrap existing controls in task containers without changing handlers
then trim wording and layout
D. Recommendation logic can become brittle

If you hardcode too many conditions from transient UI state, recommendations may open the wrong task.

Better approach

Prefer using existing workflow/routing logic if already available:

recommendWorkflowRouting(...)
snapshot-derived status
object-selection status if present
E. State reset on sample switch

When the operator changes sample/media, the app may accidentally preserve old cleanup context.

Fix

On sample change:

reset activeCleanupTask to null
optionally clear recommendation-open flags
F. Panel height creep

Even after making cleanup contextual, too many chips/subpanels can still make the left rail tall and noisy.

Fix
only one task open at a time
summary-first copy
short labels
no nested always-open subsections
