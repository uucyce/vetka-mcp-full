Issue 1: Media Bin Overwrite (Likely Owner: Beta - Import Dialog)
Probable Cause: The import handler is using array replacement instead of array append when adding new media to the bin.
Based on the Zustand patterns from the architecture and common React state bugs , the bug likely manifests as:
TypeScript
Copy

Backend File Handling: The FastAPI backend may also be generating non-unique filenames or using the same UUID for each import, causing filesystem overwrites . Production-grade file uploads must use uuid.uuid4() prefixes to prevent collisions.
Issue 2: Auto-Timeline Population (Likely Owner: Gamma - Timeline Logic)
Probable Cause: The import success handler contains a side effect that auto-creates a timeline clip immediately after media analysis completes.
In professional NLEs (Premiere/FCP7/Resolve), the import-to-timeline flow is explicitly separated :
File → Import → Media: Adds to bin only
File → Open: Opens project files (not media)
Drag from bin → timeline: Explicit user action
The current VETKA implementation appears to conflate "import success" with "add to active sequence," violating the standard three-point editing paradigm documented in the VETKA CUT Manual.
