VULNERABILITIES & RISKS
─────────────────────────────────────────────────────────────────────────────

1. Tools Disabled in Fallback ⚠️
   Severity: MEDIUM
   Location: orchestrator_with_elisya.py:1036
   Risk: When OpenRouter fallback triggered, tools=None
   Impact: Dev/QA agents lose tool calling in error scenarios
   Recommendation: Test Dev/QA functionality with OpenRouter fallback

2. XAI Key Check Not Comprehensive ⚠️
   Severity: LOW
   Location: orchestrator_with_elisya.py:1248-1250
   Risk: Only checks if key exists, not if valid
   Impact: May attempt XAI call with expired/invalid key
   Recommendation: Add key validation check before XAI call

3. Model Override Concurrency ⚠️
   Severity: LOW
   Location: orchestrator_with_elisya.py:2302
   Risk: model_routing dict modified without mutex in concurrent calls
   Impact: Potential race condition in multi-role group chats
   Recommendation: Use threading.Lock for model_routing updates

4. No Fallback Metrics ⚠️
   Severity: LOW
   Location: orchestrator_with_elisya.py:1027
   Risk: Fallback events not tracked/logged to metrics
   Impact: No visibility into fallback frequency
   Recommendation: Add telemetry on fallback events
