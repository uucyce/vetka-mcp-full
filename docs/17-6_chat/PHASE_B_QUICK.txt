### What It Does
Classifies queries to determine optimal routing:
- **"write a function"** → Dev only (fast ⚡)
- **"test this"** → QA only (focused ✅)
- **"design system"** → PM only (planning 📐)
- **"implement feature"** → PM → Dev → QA (complete 🔄)

### How It Works
1. **Heuristics**: Quick keyword matching (no LLM)
2. **LLM**: Uses llama3.2:1b for ambiguous cases
3. **Fallback**: Returns FULL_CHAIN if unsure (safe default)
