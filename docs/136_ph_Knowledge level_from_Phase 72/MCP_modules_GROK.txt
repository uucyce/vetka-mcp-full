### Моя оценка твоей идеи

**Сильные стороны:**
- Разделение ответственности = производительность и стабильность.
- Возможность запускать тяжёлые режимы **на отдельном сервере** (или даже на отдельной машине пользователя) — это огромный плюс для пользователей с мощным железом.
- Масштабируемость: можно продавать "Pro Modes" (видео-генерация, 3D-агенты) как отдельные MCP-модули.
- Архитектурно красиво: Core (VETKA) + Modes как плагины.

**Риски / слабые места:**
- Если делать слишком много отдельных MCP — сложность управления (синхронизация состояния, авторизация, UI consistency).
- Knowledge Mode ты правильно выделил — он **не тяжёлый**, его можно оставить в основном процессе.
- 3D + agents в Blender-подобном окне — это **очень тяжёлая** задача (GPU, realtime rendering, agent orchestration). Здесь отдельный MCP почти обязателен.

### Предлагаемая архитектура (2026+)

Я назвал её **"VETKA Core + Modes"**.

```
VETKA Core (Tauri)
├── Mycelium Command Center (DAG, pipelines, agents)
├── Knowledge Mode (3D graph, semantic search)
├── Directed Mode (текущий 3D)
├── Mode Switcher (узел-папка / toolbar)
│
├── Mode Plugins (опционально)
    ├── Video Generation Mode → MCP Video (локальный или remote)
    ├── 3D Agent Mode (Blender-like) → MCP 3D (с GPU)
    ├── Audio/Video Editing Mode → MCP Media
    ├── Music Generation Mode → MCP Music
    └── ... (пользовательские режимы)
```

**Как это будет работать:**

- **Core** — всегда лёгкий, отвечает за UI, 3D-граф, Mycelium, Knowledge Mode.
- **Modes** — это **отдельные процессы / MCP-серверы**.
- Переключение режима = Core говорит: "Загрузи Mode X" → либо запускает локальный MCP, либо подключается к remote.
- Коммуникация между Core и Mode — через **MCP protocol** (уже есть) + WebSocket.
- Пользователь может выбрать: "Запускать Video Mode локально" или "На моём сервере в облаке".

### Конкретные рекомендации по режимам

| Режим                    | Нужен отдельный MCP? | Почему | Где запускать |
|--------------------------|----------------------|------|---------------|
| Mycelium Command Center  | Нет                  | Лёгкий | В Core        |
| Knowledge Mode           | Нет                  | Средний | В Core        |
| 3D Agent Mode (Blender)  | **Да**               | GPU + realtime + agents | Отдельный MCP |
| Video Generation         | **Да**               | Модели (Flux, Kling, Runway) | Отдельный MCP |
| Audio Generation         | **Да**               | Тяжёлые модели | Отдельный MCP |
| Video Editing            | **Да**               | FFmpeg + AI | Отдельный MCP |

### Как технически реализовать

1. **Mode Registry** (в Core)
   - JSON-файл или база, где описано: название режима, иконка, требуемый MCP, минимальные ресурсы.

2. **MCP Launcher**
   - Tauri может запускать отдельные процессы (`tauri-plugin-shell`).
   - Или подключаться к уже запущенному remote MCP по адресу.

3. **Общий протокол**
   - Все MCP общаются с Core через один и тот же протокол (MCP JSON-RPC + WebSocket).
   - Core просто говорит: "Mode: VideoGeneration, task: generate clip from prompt".
