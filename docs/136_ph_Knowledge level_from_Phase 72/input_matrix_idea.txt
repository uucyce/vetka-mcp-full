РАЗНЫЕ ТИПЫ КОНТЕНТА → РАЗНЫЕ "ЗАВИСИМОСТИ"
Тип контента
Что является "связью" / "зависимостью"
Код
import, require, include
Видео
Таймкоды, главы, ссылки на другие видео, теги
Аудио/Подкасты
Упоминания, темы, гости, серии
Книги
Главы, ссылки на источники, цитаты, оглавление
Сценарии
Сцены, акты, персонажи, локации, арки
Документы
Ссылки, footnotes, разделы, версии
Статьи
Категории, теги, связанные статьи


ЗНАЧИТ Phase 72 — НЕ ТОЛЬКО ПРО IMPORTS
Правильная формулировка:
Phase 72 = Smart Dependency/Relationship Extraction — извлечение РЕАЛЬНЫХ СВЯЗЕЙ из контента, специфичных для типа данных.
Для кода это imports. Для видео — таймкоды и теги. Для книг — оглавление и ссылки.

АРХИТЕКТУРНЫЙ ВОПРОС
Тогда сканер должен быть модульным:
Scanner (базовый)
    │
    ├── CodeScanner      → извлекает imports
    ├── VideoScanner     → извлекает таймкоды, главы
    ├── AudioScanner     → извлекает темы, упоминания
    ├── BookScanner      → извлекает главы, ссылки
    ├── ScriptScanner    → извлекает сцены, персонажей
    └── DocumentScanner  → извлекает разделы, ссылки

Каждый возвращает унифицированную структуру связей, которую Sugiyama может использовать для построения DAG.

УТОЧНЯЮЩИЙ ВОПРОС
Phase 72 — это:
(A) Начать с кода (imports), потом расширить на другие типы?
(B) Сразу проектировать универсальную систему извлечения связей для разных типов?
(C) Что-то другое?

ВРЕМЯ КАК УНИВЕРСАЛЬНЫЙ "ИМПОРТ"
Документ (created: Jan 1)
    │
    │  ← ВРЕМЕННАЯ СВЯЗЬ (причина → следствие)
    │
    ▼
Код по документу (created: Jan 15)

Если A создан РАНЬШЕ чем B, и B семантически похож на A → A является "импортом" для B!
Это работает для ВСЕХ типов данных:
Раньше
Позже
Связь

Код

Книга
Конспект
Конспект "импортирует" книгу
Лекция (видео)
Заметки

Черновик сценария
Финальный сценарий

Исследование
Статья
Статья "импортирует" исследование


УНИВЕРСАЛЬНАЯ ФОРМУЛА СВЯЗИ
СВЯЗЬ (A → B) существует если:

1. ЯВНЫЙ ИМПОРТ (для кода)
   import A from './A'

2. ВРЕМЕННОЙ ИМПОРТ (для всего)
   created(A) < created(B) 
   AND 
   semantic_similarity(A, B) > threshold

3. ССЫЛОЧНЫЙ ИМПОРТ (для документов)
   B содержит ссылку на A


ЧТО ТАКОЕ УЗЕЛ ТЕПЕРЬ?
Узел = точка, из которой "растут" зависимые сущности
В коде: main.py, __init__.py, hub-файлы
В документах: первоисточник (то, что было создано раньше и породило остальное)
В видео: оригинальное видео (от которого пошли нарезки, конспекты, обсуждения)

Y-ОСЬ = ВРЕМЯ + KNOWLEDGE LEVEL
Тогда Y-ось в Knowledge Mode это:
Y = f(время_создания, knowledge_level)

Внизу (Y=0): Первоисточники, базовые концепты, ранние документы
Вверху (Y=1): Производные, сложные концепты, поздние версии

Причинно-следственная иерархия!
