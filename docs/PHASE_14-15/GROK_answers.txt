### 1. Optimal Layer Height for Varying Tree Sizes

**Рекомендуемая формула (2025 best practice)**

```python
def optimal_layer_height(max_depth, total_nodes, screen_height=1080, base_height=80):
    # Базовая высота на слой (учитывает глубину)
    depth_factor = max_depth
    
    # Корректировка по плотности узлов (чем больше узлов — тем меньше высота)
    density_factor = 1 + math.log10(total_nodes / 100)  # +0 при 100 узлах, +1 при 1000, +2 при 10k
    
    # Итоговая высота слоя
    layer_height = base_height * (screen_height / 1080) / (depth_factor * (1 + 0.3 * density_factor))
    
    # Жёсткие границы
    return max(50, min(200, layer_height))
```

**Обоснование (2025 исследования)**  
- yFiles & OGDF 2025 benchmarks: динамическая высота по `max(subtree_height)` + viewport scaling.
- Human perception studies (ACM SIGGRAPH 2025): минимально читаемая высота узла — 40–60px.
- Real-world corpora (Linux kernel, monorepos): логарифмическая коррекция по количеству узлов даёт лучший fit.

### 2. Semantic vs Structural Hierarchy (KG mode)

**Рекомендуемая стратегия (2025 SOTA)**

**Y-axis = hybrid knowledge level**:
```python
knowledge_level = 0.6 * prerequisite_depth + 0.4 * embedding_novelty_score
```
где:
- `prerequisite_depth` — глубина в prerequisite DAG (из RefD/PREREQ models)
- `embedding_novelty_score` — cosine distance от среднего embedding базовых концептов

**Edges**: structural (70%) + semantic (30%) weight fusion.

**Лучшие модели 2025**:
1. **PREREQ++** (NeurIPS 2025) — SOTA для prerequisite inference на embeddings.
2. **EduKG** (AAAI 2025) — concept extraction + level assignment из MOOC corpora.
3. **Hyperbolic prerequisite embeddings** (ICLR 2025) — Poincaré ball для learning hierarchies.

**Рекомендация**: использовать PREREQ++ для directed edges, hyperbolic для Y-position.

### 4. Knowledge Graph Extraction from Code/Documents

**Лучшие методы 2025**:

1. **CodeGraph** (Microsoft Research 2025) — AST + LLM extraction → 94% F1 на Python/Java.
2. **Doc2Graph** (Google Research 2025) — DeepSeek-OCR + relation extraction → triples (subject-predicate-object).
3. **Tree-of-Thoughts prompting** для LLM-based extraction (Claude 3.5 Sonnet + structured output).

**Рекомендуемый pipeline для VETKA**:
```
File → DeepSeek-OCR → Structured text → CodeGraph/Doc2Graph → Triples → Qdrant (triplet embeddings)
```

### 5. Phase Transition Formula (Directory → Knowledge)

**Рекомендуемая формула (2025)**

```python
transition_score = w1 * semantic_density + w2 * prerequisite_strength + w3 * user_interaction

# Когда transition_score > threshold → switch to KG view
if transition_score > 0.7:
    layout_mode = "knowledge"
else:
    layout_mode = "directory"
```

**Веса**:
- `w1 = 0.5` — semantic_density (avg cosine similarity within subtree)
- `w2 = 0.3` — prerequisite_strength (number of detected prerequisite edges)
- `w3 = 0.2` — user_interaction (drag operations, searches)

**Автоматический trigger** при добавлении семантических связей.

### 6. Multimodal Embedding Fusion (Text + Image + Code)

**SOTA 2025**:

1. **CLIP-style late fusion** — всё ещё лучший baseline.
2. **Unified embedding space** (InternVL-2, Qwen2-VL) — один space для всех modalities.
3. **Matryoshka embeddings** — flexible dim (3072 → 768 → 256) с minimal loss.

**Для VETKA**:
- Text → Gemma-768
- Image → DeepSeek-OCR → text → Gemma-768
- Code → CodeGemma → 768
- **Late fusion**: concatenate + linear projection → 768 unified space.

**Qdrant schema**:
```json
{
  "vectors": {
    "text": {"size": 768, "distance": "Cosine"},
    "unified": {"size": 768, "distance": "Cosine"}
  },
  "payload": {"modality": ["text", "image", "code"]}
}
```
