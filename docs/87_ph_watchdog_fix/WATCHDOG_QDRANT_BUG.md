# Phase 87: Watchdog → Qdrant Bug Report

**Дата:** 2026-01-21
**Статус:** ✅ FIXED

## Проблема
Watchdog детектирует изменения файлов, но НЕ индексирует в Qdrant.

Логи терминала показывают:
```
21:22:48 watchfiles.main - INFO - 1 change detected
21:22:48 watchfiles.main - INFO - 3 changes detected
```

Но файлы не появляются в VETKA tree и поиске.

## Корневая причина

**files_routes.py:400** вызывает `get_watcher()` БЕЗ `qdrant_client`:

```python
# src/api/routes/files_routes.py:400
watcher = get_watcher()  # ← qdrant_client=None!
```

Если это ПЕРВЫЙ вызов `get_watcher()`, singleton создаётся с `qdrant_client=None`.

Логика в `file_watcher.py:504-511`:
```python
if _watcher_instance is None:
    _watcher_instance = VetkaFileWatcher(socketio=socketio, qdrant_client=qdrant_client)
    _watcher_instance.load_state()
else:
    # Lazy injection - но уже поздно если watcher запущен
    if socketio and _watcher_instance.socketio is None:
        _watcher_instance.socketio = socketio
    if qdrant_client and _watcher_instance.qdrant_client is None:
        _watcher_instance.qdrant_client = qdrant_client
```

## Решение

1. **Вариант A:** Добавить qdrant_client в files_routes.py
2. **Вариант B:** Принудительная инъекция qdrant_client при старте сервера
3. **Вариант C:** Создать endpoint для ре-инициализации watcher с qdrant
