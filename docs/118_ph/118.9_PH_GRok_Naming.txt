## **3. Naming Philosophy: Мои мысли (Balance Unique ID vs. Intuitive Search)**
Ты прав — **уникальность ID vs. интуитивность** критично для MCP/tools (LLM не "вспоминает" без подсказок). Текущий хаос (mycelium/dragon/titan/spawn) путает, но **иерархия возможна** без потери UX.

**Проблемы текущих имён:**
- "spawn" — game-like, не VETKA-style.
- "myc_out" — аббревиатура (mycelium?), но "myc" может конфликтовать с mycology libs или "Myc" agents.
- Много имён: spawn/mycelium/dragon/doctor/titan — **слишком!** → Унифицировать под **VETKA prefix**.

**Моё предложение: "vetka_out" + Иерархическая Лига**
| **Категория** | **Naming Rule** | **Примеры** | **Почему?** |
|---------------|-----------------|-------------|-------------|
| **Директории/Output** | `vetka_<type>_out` | `vetka_myc_out/`, `vetka_spawn_out/` (если нужно), `data/vetka_staging/` | Prefix для grep/search, intuitive (vetka = core brand). |
| **Tools/MCP** | `vetka_<domain>_<action>` | `vetka_cam_calculate_surprise`, `vetka_elisya_context`, `vetka_artifact_edit` | Unique ID (vetka_*), intuitive (cam/elisya/artifact domains). LLM легко "вспомнит" по prefix. |
| **Pipelines/Agents** | `vetka_<league>_<tier>` | `@vetka_dragon_bronze`, `@vetka_titan_prime`, `@vetka_doctor_fix` | Лиги (Dragon=cheap Asian, Titan=premium) — из Titans League doc (pinned). Doctor=debug. |
| **Internal Vars** | Short + descriptive | `MAX_PARALLEL`, `STM_RESET_MAX` (твой план ✅) | Экономия токенов в ELISION. |
| **ELISION** | Auto-short | `"vetka_mycelium_pipeline": "vmp"` | Не трогаем код — сжимаем в промптах. |

**Преимущества:**
- **Search-friendly:** `vetka_*` — 1 grep для всего.
- **LLM-intuitive:** Prompts: "Use vetka_cam_* tools for CAM, vetka_artifact_* for edits."
- **Harmony:** Всё под VETKA (не mycelium/spawn как foreign). Mycelium=internal pipeline name (docs), но dirs=vetka_out.
- **Scale:** Легко добавить `vetka_tavily_search`, `vetka_context7_docs`.

**Альтернатива "myc_out":** Если mycelium=core theme (из Mycelium Audit), то `vetka_myc_out/` — но "vetka_out/" проще (экономит chars).
