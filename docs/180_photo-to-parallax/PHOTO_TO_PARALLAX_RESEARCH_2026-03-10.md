# Photo-to-Parallax Research

Дата фиксации: `2026-03-10`

## 1. Контекст задачи

Цель инструмента: из одного изображения получать набор ассетов и рендерить мягкий параллакс-ролик без тяжёлой ручной компоновки.

Практический use case:

- друг пользователя регулярно делает "живые" фото-видео;
- движение камеры небольшое;
- критично быстро получать чистые выходные ассеты;
- важнее стабильность пайплайна, чем научная новизна depth-модели.

Ключевой вывод: продукт надо проектировать не как `depth map generator`, а как `photo-to-parallax asset pipeline`.

Финальные артефакты пайплайна:

- `depth_16.png`
- `depth_preview.png`
- `subject_mask.png`
- `subject_rgba.png`
- `clean_plate.png`
- `overscan_plate.png`
- `shot_preview.mp4`
- `metadata.json`

## 2. Этап 1: Depth map и стабилизация диапазона

### Цель

Получить depth-карту, пригодную не только для визуализации, но и для последующих решений:

- выделение ближнего слоя;
- оценка disocclusion risk;
- подсказка для сегментации;
- предварительное разнесение слоёв в 2.5D.

### Что реально нужно

Для параллакса depth должна быть:

- стабильной на границах объектов;
- контрастной между foreground и background;
- достаточно гладкой внутри больших поверхностей;
- воспроизводимой между разными изображениями.

### Что не работает как единственная стратегия

- простой `autocontrast`;
- один фиксированный threshold для всех сцен;
- использование только 8-bit preview depth без 16-bit master depth.

### Рекомендуемый пайплайн нормализации

1. получить `raw depth`;
2. сохранить `16-bit master`;
3. выполнить `percentile normalization`, например `p2-p98`;
4. применить мягкое edge-aware сглаживание;
5. отдельно построить `preview depth` через gamma/contrast;
6. только потом пробовать `threshold`, `Otsu` или `k-means`.

### Кандидаты depth backend

#### `Depth Anything V2 Small`

Почему силён:

- хороший баланс качества и скорости;
- Small-модель подходит как pragmatic MVP backend;
- удобный Python inference.

Ограничения:

- depth в основном относительный;
- для агрессивного автоматического matte всё равно нужен refinement.

Источник:

- [Depth Anything V2](https://github.com/DepthAnything/Depth-Anything-V2)

#### `Depth Pro`

Почему особенно интересен:

- явный фокус на sharp boundaries;
- metric depth может быть полезен для более предсказуемой геометрии;
- хорош как кандидат для bake-off именно под parallax-задачу.

Ограничения:

- стек тяжелее;
- требуется отдельная проверка скорости и устойчивости на реальных пользовательских картинках.

Источник:

- [Depth Pro](https://github.com/apple/ml-depth-pro)

#### `MiDaS`

Плюсы:

- зрелый baseline;
- полезен для sanity check и сравнения.

Минусы:

- репозиторий архивирован;
- как основной будущий стек выглядит слабее новых кандидатов.

Источник:

- [MiDaS](https://github.com/isl-org/MiDaS)

### Вывод по этапу 1

Для MVP и исследований надо делать сравнение не `Depth Anything V2 vs MiDaS`, а:

- `Depth Anything V2 Small`
- `Depth Pro`

Именно они должны пройти bake-off на локальном наборе тестовых изображений.

## 3. Этап 2: Выделение объекта и построение чистого clean plate

### Цель

После удаления foreground получить фон без дырок на месте вырезанных объектов.

### Важное различие

Это не то же самое, что overscan/outpaint за пределы исходного кадра.

Этап 2 решает:

- дырки в центре сцены;
- за объектом или человеком;
- артефакты в области удалённой маски.

### Практический pipeline

1. получить coarse mask из depth prior;
2. расширить маску на несколько пикселей;
3. подчистить края морфологией;
4. запустить large-mask inpainting;
5. при необходимости добить мелкие дефекты классическим inpaint.

### Инструменты-кандидаты

#### `SAM 2`

Роль:

- refinement маски;
- улучшение границ после грубой depth-guided сегментации.

Источник:

- [SAM 2](https://github.com/facebookresearch/sam2)

#### `LaMa`

Роль:

- основной backend для large-mask inpainting;
- хорошо подходит для удаления крупных объектов на фото.

Источник:

- [LaMa](https://github.com/advimman/lama)

#### OpenCV inpaint

Роль:

- локальная доводка маленьких дефектов;
- швы, мелкие дыры, неровные края после основного inpaint.

### Вывод по этапу 2

Для MVP clean plate должен строиться так:

- `depth-guided mask`
- `mask dilation`
- `LaMa`
- `OpenCV inpaint` как optional cleanup

## 4. Этап 3: Overscan plate

### Цель

Создать изображение больше исходного кадра, чтобы мягкое движение виртуальной камеры не открывало пустые края.

### Что решает этот этап

- выход камеры за границы исходного холста;
- лёгкий zoom out / dolly;
- дополнительный запас за спиной foreground.

### Что он не решает

- дырки на месте вырезанного человека сами по себе;
- плохую маску foreground.

### Практический pipeline

1. взять `clean_plate.png`;
2. расширить canvas на `8-20%`;
3. построить band-mask по внешнему краю;
4. выполнить outpaint;
5. сохранить `overscan_plate.png`.

### Вывод по этапу 3

`clean plate` и `overscan plate` должны существовать как разные сущности.

Минимальный MVP:

- одно изображение фона после удаления объекта;
- поверх него optional overscan-режим;
- отдельные метрики покрытия по краям.

## 5. Этап 4: Разнесение слоёв в 2.5D

### Цель

Создать ощущение объёма без настоящей 3D-реконструкции.

### Реалистичный старт

`2 layers` достаточно только для очень умеренного движения.

Риски:

- cardboard effect;
- halo на границе foreground;
- раскрытие дыр за объектом при слишком большом travel.

### Рекомендуемая стратегия слоёв

#### MVP

- `foreground subject`
- `background plate`

#### Следующий шаг

- `foreground`
- `midground`
- `background`

### Практические ограничения для двух слоёв

- горизонтальный travel около `2-4%` ширины кадра;
- вертикальный travel ещё меньше;
- zoom обычно `1.02-1.08`;
- сильное движение сразу повышает risk score.

### Вывод по этапу 4

Для первого релиза допустимы `2 слоя`, но архитектура должна сразу допускать `3-5 слоёв`.

## 6. Этап 5: FFmpeg merge/render

### Цель

Быстро собрать итоговый ролик из подготовленных ассетов.

### Почему `ffmpeg` подходит

- хорошо анимирует 2D-плоскости;
- даёт repeatable CLI render;
- удобно автоматизировать из Python/bash;
- достаточен для mild parallax.

### Что не стоит перекладывать на `ffmpeg`

- сегментацию;
- inpainting;
- логику построения clean plate;
- выбор motion preset по сцене.

### Что должен делать renderer

- читать `subject_rgba`, `overscan_plate`, `metadata.json`;
- строить filtergraph;
- анимировать background и foreground с разной скоростью;
- собирать `preview.mp4` и финальный render.

Источник:

- [FFmpeg filter documentation](https://patches.ffmpeg.org/ffmpeg-filters.html)

## 7. Сравнение вариантов по этапам

| Этап | MVP-кандидат | Почему |
| --- | --- | --- |
| Depth | `Depth Anything V2 Small` и `Depth Pro` | Самые полезные кандидаты для bake-off |
| Coarse mask | depth threshold + clustering | Дёшево и быстро |
| Refine mask | `SAM 2` | Лучше края и сложные формы |
| Clean plate | `LaMa` | Large-mask inpainting |
| Small cleanup | OpenCV inpaint | Дешёвая локальная правка |
| Overscan | тот же inpaint/outpaint backend | Не нужен отдельный продуктовый остров |
| Render | `ffmpeg` | Детерминированный финальный merge |

## 8. Риски

### Технические

- relative depth не даёт стабильной абсолютной геометрии;
- прозрачные, блестящие и тонкие объекты плохо режутся;
- большие движения камеры быстро ломают illusion.

### Продуктовые

- если автоматическая маска плохая, пользователь всё равно вернётся к ручной правке;
- если overscan не отделён от clean plate, будет путаница в UX и артефактах.

### Лицензионные

- нужно избегать неоднозначных non-commercial условий;
- для продукта безопаснее держаться `Depth Anything V2 Small`, а не более тяжёлых вариантов без явной совместимости.

## 9. Рекомендация по MVP

Порядок сборки MVP:

1. `Depth bake-off`: `Depth Anything V2 Small` vs `Depth Pro`.
2. `Mask pipeline`: adaptive threshold + optional `SAM 2`.
3. `Clean plate`: `LaMa`.
4. `Overscan`: отдельный этап от `clean plate`.
5. `Renderer`: `ffmpeg`.

## 10. Рекомендация по исследовательской песочнице

Песочница должна проверять не только красивость превью, а эксплуатационные вопросы:

- насколько велик требуемый overscan;
- где ожидается disocclusion;
- когда `2 layers` уже недостаточно;
- какой motion preset безопасен для конкретной картинки;
- насколько source image годится под mild parallax вообще.
