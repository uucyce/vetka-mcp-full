# MCC Drill-Down Mechanics Synthesis (2026-02-27)

Статус: Research synthesis only (без смены стека, без миграции библиотек).
Источник: разбор MACE-подхода + historical drill-down паттерны + текущий MCC контекст.

## 0) Цель

Зафиксировать рабочую механику для MCC в рамках одного окна:
- один canvas;
- прозрачная иерархия;
- фрактальное раскрытие узлов/тасков;
- без хаотичного глобального relayout;
- без потери контекста и зависимостей.

## 1) Drill-Down Contract (toggle expand/collapse)

Принцип:
- раскрытие/сворачивание делается внутри того же canvas, без второго окна.

Контракт событий:
- single click: выделение узла + связи (focus signal).
- double click (или Enter на выделенном): toggle раскрытия подграфа узла.
- click по пустому месту: снять выделение, не ломать состояние раскрытых узлов.

Ограничения:
- без автоперехода в другой режим/экран;
- без side effects, меняющих root-level layout целиком.

Done-критерий:
- любой узел раскрывается/сворачивается в текущем canvas предсказуемо и обратимо.

## 2) Focus + Context

Принцип:
- подграф раскрывается локально вокруг якорного узла;
- остальной DAG остается видимым как контекст.

Правила:
- anchor node всегда остается в кадре и является точкой происхождения раскрытия;
- workflow/task-subgraph визуально отделяется размером/стилем, но остается связанным с anchor;
- глобальная прозрачность всего графа не применяется как default.

Done-критерий:
- после раскрытия пользователь видит одновременно: anchor, раскрытый подграф, и основной контекст дерева.

## 3) Auto-Collapse + Threshold

Принцип:
- не показывать весь объем дочерних структур сразу.

Правила порогов:
- depth cap: по умолчанию 1-2 уровня за одно раскрытие;
- breadth cap: ограничение количества детей в первом кадре;
- overflow marker: "+N" для нераскрытой части;
- повторный expand открывает следующую порцию/уровень.

Done-критерий:
- нет визуальной "каши" при узлах с большим количеством связей.

## 4) Breadcrumb / Path

Принцип:
- пользователь всегда понимает, где он находится в иерархии.

Правила:
- верхняя строка пути: Root > ... > Parent > Current;
- click по сегменту пути сворачивает глубину до выбранного сегмента;
- path одинаков для task-drill и folder-drill.

Done-критерий:
- нет потери контекста при 2+ шагах раскрытия.

## 5) Lazy Unfold (on-demand)

Принцип:
- подграф строится/подгружается только при explicit expand.

Правила:
- первичный DAG не содержит все внутренние workflow-узлы заранее;
- при collapse временные inline-узлы убираются;
- повторный expand воспроизводим (детерминированная сборка).

Done-критерий:
- стабильный FPS/отклик, отсутствие "подземных улетов" и накопления мусорного состояния.

## 6) Relayout Only Local

Принцип:
- перераскладка только в зоне раскрытия, а не во всем графе.

Правила:
- локальный bounding region вокруг anchor;
- push-away применяется только к узлам, реально пересекающим region;
- корневые слои и дальние ветки не пересчитываются;
- после collapse локальный push откатывается.

Done-критерий:
- нет глобального скачка камеры/слоев;
- соседние несвязанные ветки не "улетают".

## 7) Специфика для Task -> Workflow (MCC)

Контракт:
- workflow рождается из task-node (anchor link обязателен);
- workflow-узлы меньше архитектурных (фрактальный scale);
- workflow имеет свою внутреннюю DAG-форму и не должен смешиваться с глобальной топологией;
- при наличии места раскрытие вниз/вверх по контракту ориентации проекта (bottom-root invariant сохраняется).

Done-критерий:
- визуально читаемая мини-DAG структура workflow рядом с таском, а не хаотичный зигзаг/полоса.

## 8) Специфика для Folder/Code Nodes

Контракт:
- раскрытие папок/модулей тем же drill-механизмом;
- сначала children первого уровня, затем grandchildren по повторному toggle/step;
- показывать счетчик содержимого (files/subdirs/tasks) в ноде;
- не создавать "одиночные бессмысленные" вылеты по одной ноде, если доступна минимальная группа.

Done-критерий:
- folder-drill дает информативный срез, а не случайный single-node выброс.

## 9) Anti-Patterns (что исключаем)

- Глобальный force-relayout при каждом drill.
- Автоматический zoom-jump как обязательный побочный эффект раскрытия.
- Открытие workflow в отдельном экране/окне при inline-сценарии.
- Переезд на новый стек как "фикс механики" вместо локального контракта состояния/лейаута.

## 10) Минимальный acceptance-checklist

1. Double click по task: workflow раскрыт inline в том же canvas.
2. Повторный double click: workflow аккуратно свернут.
3. Empty click: снимает выделение, не рушит drill-state.
4. Раскрытие не вызывает глобальный скачок камеры.
5. Раскрытие не делает весь остальной граф полупрозрачным по умолчанию.
6. Большой узел раскрывается порционно (threshold работает).
7. Breadcrumb отражает текущий путь и позволяет drill-up.
8. Folder drill показывает группу содержимого, а не случайную одиночную ноду.

## 11) Вывод

Да, исследование Грока полезно не библиотеками, а механическими инвариантами:
- toggle drill contract,
- focus+context,
- threshold unfold,
- breadcrumb,
- lazy unfold,
- local-only relayout.

Этого достаточно, чтобы обновить архитектурный контур MCC в рамках текущего стека и вашей концепции "все в одном окне".
