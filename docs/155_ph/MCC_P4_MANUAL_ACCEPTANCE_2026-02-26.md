# MCC P4 Ручная Приемка (2026-02-26)

Дата: 2026-02-26  
Охват: `MARKER_155.P4_1.FOCUS_ACTION_PARITY.V1`, `MARKER_155.P4_2.FOCUS_MEMORY.V1`, `MARKER_155.P4_3.FOCUS_DISPLAY_MODES.V1`  
Ответственный: MCC UX/Runtime

## Цель

Закрыть G1 свежим ручным прогоном приемки для UX-поведения P4:
- сохранение фокуса при zoom/drill,
- поведение Architect prefill,
- стабильность режимов отображения фокуса.

## Предусловия

- Приложение запущено с MCC UI.
- Roadmap DAG загружен для реального project scope.
- MiniChat виден.

## Чеклист

1. Multi-select в DAG через Shift-click:
- Выбрать 2+ узла с зажатым Shift.
- Ожидается: focus set растет аддитивно, без сброса на каждом клике.

2. Поведение режимов отображения фокуса:
- Переключать `all`, `selected+deps`, `selected-only`.
- Ожидается: фильтрация соответствует семантике режима, после переключения не остается «зависших» скрытых узлов.

3. Сохранение при zoom/drill:
- С активным focus set выполнить zoom in/out и drill-переходы.
- Ожидается: валидные focused ids сохраняются, устаревшие ids безопасно отсекаются.

4. Паритет Architect prefill:
- Вызвать `Shift+Enter` из состояния с выбранным фокусом.
- Нажать кнопку "Ask Architect" в том же focus-контексте.
- Ожидается: оба действия заполняют MiniChat эквивалентным payload выбранного scope.

5. Проверка на регресс:
- Очистить focus.
- Сформировать focus set заново.
- Повторить шаги 2-4.
- Ожидается: нет «залипших» состояний, нет пустого prefill при непустом focus set.
