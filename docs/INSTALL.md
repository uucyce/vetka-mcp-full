# 🚀 VETKA Live 0.3 - Профессиональная установка

## 🎯 Установка в изолированной среде

### Быстрый запуск установщика:

```bash
python install_vetka.py
```

## 🎮 Что делает установщик:

### ✅ Автоматически:
1. **Выбор пути установки** - спрашивает где создать VETKA
2. **Проверка системы** - проверяет Python, место на диске
3. **Изолированное окружение** - создает отдельный .venv
4. **Установка зависимостей** - по группам, с обработкой ошибок
5. **Копирование файлов** - все файлы проекта в новое место
6. **Конфигурация** - создает .env с настройками
7. **Лаунчеры** - создает start_vetka.sh/.bat

### 🏗️ Структура установки:

```
[Выбранный путь]/VETKA/
├── .venv/                    # Изолированное виртуальное окружение
├── .env                      # Конфигурация
├── vetka_config.json         # Информация об установке
├── start_vetka.sh/.bat       # Лаунчеры
├── main.py                   # Основное приложение
├── config/                   # Конфигурация системы
├── src/                      # Исходный код
├── frontend/                 # 3D интерфейс
├── tests/                    # Тесты
└── ...                       # Все файлы проекта
```

## 🎯 Преимущества изолированной установки:

- ✅ **Никаких конфликтов** с другими проектами
- ✅ **Отдельные зависимости** для каждого компонента
- ✅ **Портативная установка** - можно перемещать
- ✅ **Легкое удаление** - просто удалить папку
- ✅ **Несколько версий** - можно установить разные версии
- ✅ **Безопасность** - не затрагивает системный Python

## 🚀 После установки:

### Запуск:
```bash
cd [путь установки]/VETKA
./start_vetka.sh          # macOS/Linux
start_vetka.bat           # Windows
```

### Настройка:
1. Откройте `.env` файл
2. Добавьте свои API ключи:
   - `OPENROUTER_KEY_1=ваш_ключ`
   - `GEMINI_API_KEY=ваш_ключ`
3. Сохраните файл

### Первый запуск:
1. Запустите лаунчер
2. Откройте http://localhost:5001
3. Попробуйте команды:
   - `/bot/create API`
   - `/bot/analyze code`

## 🔧 Управление установкой:

### Переустановка:
```bash
python install_vetka.py
# Выберите тот же путь - старые файлы будут заменены
```

### Обновление:
```bash
# В директории установки
source .venv/bin/activate
pip install --upgrade -r requirements.txt
```

### Удаление:
```bash
rm -rf [путь установки]/VETKA
```

## 🐛 Решение проблем:

### Конфликт портов:
- Weaviate: 8080
- Ollama: 11434
- VETKA FastAPI: 5001

Если порты заняты, измените в `.env`:
```bash
WEAVIATE_URL=http://localhost:8081
OLLAMA_URL=http://localhost:11435
FLASK_PORT=5001
```

### Проблемы с зависимостями:
Установщик автоматически обрабатывает проблемы:
- Пробует установить точную версию
- Если не получается - устанавливает последнюю
- Логирует все проблемы

### Проблемы с Python 3.13:
Установщик тестирован с Python 3.13 и обрабатывает специфичные проблемы.
