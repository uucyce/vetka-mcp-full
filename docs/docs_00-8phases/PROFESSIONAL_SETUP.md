# 🌳 VETKA Live 0.3 - Профессиональная установка

## 🎯 Решение проблемы конфликтов

Проблема с конфликтами портов и зависимостей решена! Теперь VETKA устанавливается в **полностью изолированную среду**.

## 🚀 Установка в один клик:

### Рекомендуемый способ:
```bash
python install_vetka.py
```

### Или через скрипты:
```bash
./setup.sh          # macOS/Linux
setup.bat           # Windows
```

## 🎮 Что происходит при установке:

### 1. 📁 Выбор пути установки
- Установщик спрашивает где создать VETKA
- Предлагает варианты: `/Users/username/VETKA`, `/opt/VETKA`, etc.
- Можно ввести свой путь

### 2. 🔍 Проверка системы
- Проверяет Python 3.8+
- Проверяет свободное место на диске
- Проверяет наличие pip

### 3. 🏗️ Создание изолированной среды
- Создает отдельный `.venv` в выбранной директории
- Полностью изолирован от системного Python
- Не конфликтует с другими проектами

### 4. 📦 Установка зависимостей по группам
- **Основной фреймворк**: Flask, SocketIO
- **LangGraph и агенты**: LangChain, CrewAI
- **Векторные базы**: Weaviate, Qdrant
- **LLM провайдеры**: OpenRouter, Ollama, OpenAI
- **Обработка файлов**: Pillow, Pygments, PDF
- **Утилиты**: HTTP, Pydantic, NumPy
- **Тестирование**: pytest

### 5. 📋 Копирование файлов проекта
- Копирует все файлы в новое место
- Создает полную структуру проекта

### 6. ⚙️ Создание конфигурации
- Создает `.env` файл с настройками
- Создает `vetka_config.json` с информацией об установке
- Настраивает порты и пути

### 7. 🚀 Создание лаунчеров
- `start_vetka.sh` для macOS/Linux
- `start_vetka.bat` для Windows
- Автоматическая активация venv

## 🏗️ Структура установки:

```
[Выбранный путь]/VETKA/
├── .venv/                    # Изолированное виртуальное окружение
├── .env                      # Конфигурация (добавьте API ключи)
├── vetka_config.json         # Информация об установке
├── start_vetka.sh/.bat       # Лаунчеры
├── main.py                   # Основное приложение
├── config/                   # Конфигурация системы
├── src/                      # Исходный код
│   ├── agents/               # VETKA агенты
│   ├── memory/               # Weaviate интеграция
│   └── workflows/            # LangGraph workflows
├── langgraph_flows/          # Workflow определения
├── elisya_integration/       # Elisya контекст
├── frontend/                 # 3D интерфейс
├── tests/                    # Тесты
└── ...                       # Все остальные файлы
```

## 🚀 После установки:

### 1. Запуск:
```bash
cd [путь установки]/VETKA
./start_vetka.sh          # macOS/Linux
start_vetka.bat           # Windows
```

### 2. Настройка API ключей:
Откройте `.env` файл и добавьте:
```bash
OPENROUTER_KEY_1=ваш_ключ_openrouter
OPENROUTER_KEY_2=ваш_ключ_openrouter_2
GEMINI_API_KEY=ваш_ключ_gemini
```

### 3. Первый запуск:
1. Запустите лаунчер
2. Откройте http://localhost:5000
3. Попробуйте команды:
   - `/bot/create API`
   - `/bot/analyze code`
   - `/visual/tree`

## 🔧 Управление установкой:

### Обновление зависимостей:
```bash
cd [путь установки]/VETKA
source .venv/bin/activate
pip install --upgrade -r requirements.txt
```

### Переустановка:
```bash
python install_vetka.py
# Выберите тот же путь - файлы будут заменены
```

### Удаление:
```bash
rm -rf [путь установки]/VETKA
```

## 🐛 Решение проблем портов:

Если порты заняты, измените в `.env`:
```bash
WEAVIATE_URL=http://localhost:8081
OLLAMA_URL=http://localhost:11435
FLASK_PORT=5001
```
