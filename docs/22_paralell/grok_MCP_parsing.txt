### TOPIC 1: MCP Server Architecture for Multi-Agent Interoperability

#### Research Findings
- Anthropic MCP (2025) — это **новый стандарт** для tool use в браузере/IDE (Computer Use API + MCP Server).  
  Документация: https://docs.anthropic.com/en/docs/build-with-claude/computer-use#mcp-server  
  Основная идея: MCP — это **WebSocket-сервер**, который агенты используют для tool calling (browse, edit_file, run_command и т.д.).  
- Multi-vendor: пока **только Anthropic** имеет MCP, но сообщество активно делает **reverse-engineered MCP** для Ollama, vLLM, OpenAI, Gemini.  
  Самые сильные open-source реализации (декабрь 2025):
  - https://github.com/steve02081504/mcp-server (TypeScript, поддержка Ollama + OpenAI) — 1.2k stars
  - https://github.com/lm-sys/mcp-bridge (Python, multi-model) — 800+ stars
  - https://github.com/continuedev/mcp-proxy (Continuedev fork, VS Code integration)

#### Recommendations for VETKA
- **Красный**: Сделать VETKA **MCP-совместимым сервером** (WebSocket endpoint /mcp) — это позволит **любому** Claude/Grok/GPT агенту сразу подключаться и работать с твоей knowledge graph.  
- Не изобретать велосипед: взять **mcp-server** (TS) или **mcp-bridge** (Python) как базу.  
- Добавить свои tools: create_branch, search_knowledge, edit_file, run_agent_task.

#### Technical Specifications
- WebSocket endpoint: ws://localhost:5001/mcp
- Protocol: JSON-RPC 2.0 over WS (Anthropic spec)
- Tools schema: OpenAI-style function calling + Anthropic tool use

#### Code Examples
```python
# src/mcp/mcp_server.py (Flask + Socket.IO)
from flask_socketio import SocketIO

socketio = SocketIO(app)

@socketio.on('connect', namespace='/mcp')
def mcp_connect():
    print('[MCP] Agent connected')

@socketio.on('tool_call', namespace='/mcp')
def handle_tool_call(data):
    tool_name = data['name']
    args = data['arguments']
    
    if tool_name == 'search_knowledge':
        result = search_knowledge(args['query'])
        socketio.emit('tool_result', {'id': data['id'], 'result': result}, namespace='/mcp')
```

### TOPIC 2: Agent Memory Transfer Protocol

#### Research Findings
- Нет единого стандарта в 2025, но самые сильные форматы:
  - **OpenAI Threads API** (JSON + messages array + tool_calls)
  - **Anthropic Messages API** (messages + system prompt + tool results)
  - **LangGraph checkpointer** (JSON + state snapshot)
  - **Mem0** (vector + graph memory export)

#### Recommendations for VETKA
- **Красный**: Создать **.vetka-mem** формат — JSON + embedded vectors + Weaviate refs  
- Bidirectional sync:  
  - VETKA → Claude Desktop: export memory → copy-paste или file drop  
  - Claude → VETKA: import thread → parse messages → create branch

#### Technical Specifications
```json
{
  "format": "vetka-memory-v1",
  "timestamp": "2025-12-28T23:45:00Z",
  "project_branch": "main",
  "system_prompt": "...",
  "messages": [
    {"role": "user", "content": "..."},
    {"role": "assistant", "content": "...", "tool_calls": [...]}
  ],
  "memory_vectors": [
    {"id": "uuid", "embedding": [0.1, ...], "text": "..."}
  ],
  "weaviate_refs": ["uuid1", "uuid2"]
}
```

#### Code Examples
```python
def export_memory(branch_id: str):
```

### TOPIC 3: Universal Parser/Intake System

#### Research Findings
- **YouTube**: yt-dlp (2025 — 2024.12.20+) — всё ещё лидер  
  Транскрипция: Whisper.cpp или local Whisper (faster-whisper)  
- **Соцсети**:  
  - X/Twitter — официальный API ($100+/мес) или Nitter/RSS (legal gray)  
  - Telegram — Telethon/Telethon-py  
  - LinkedIn/Instagram — **запрещено** без API (rate limit + ban)  
  - RSS — feedparser  
- **Браузерная история**:  
  - Chrome: sqlite3 ~/.config/google-chrome/Default/History  
  - Safari: ~/Library/Safari/History.db  
  - Privacy: **только local**, opt-in

#### Recommendations for VETKA
- **Красный**: Сделать **intake pipeline** с yt-dlp + Whisper.cpp + Telethon  
- Legal matrix:  
  - YouTube: OK (personal use)  
  - X: OK с API  
  - Telegram: OK  
  - LinkedIn/Instagram: **запрещено** без согласия

#### Technical Specifications
```python
def intake_youtube(url: str):
    info = yt_dlp.YoutubeDL().extract_info(url, download=False)
    transcript = whisper_transcribe(info['url'])
    return {"title": info['title'], "transcript": transcript}
```
