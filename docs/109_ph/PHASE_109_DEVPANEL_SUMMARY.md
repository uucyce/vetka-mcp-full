## Features Implemented

### 1. DevPanel Component (`Cmd+Shift+D`)

**Controls:**
- **Y_WEIGHT_TIME** slider (0-1) - time-based Y positioning weight
- **Y_WEIGHT_KNOWLEDGE** auto-calculated (1 - TIME)
- **MIN_Y_FLOOR** input (default: 20)
- **MAX_Y_CEILING** input (default: 5000)
- **FALLBACK_THRESHOLD** slider (0-1, default: 0.5)
- **USE_SEMANTIC_FALLBACK** toggle (default: true)

**Buttons:**
- Apply (saves to localStorage, emits event)
- Reset (clears to defaults)

### 2. Threshold-Based Fallback

**Before:**
```typescript
// All-or-nothing: ALL nodes must be (0,0,0)
const needsLayout = Object.values(allNodes).every(
  (n) => n.position.x === 0 && n.position.y === 0 && n.position.z === 0
);
```

**After:**
```typescript
// Threshold-based: triggers when invalid ratio exceeds threshold
const invalidRatio = invalidCount / totalNodes;
const needsLayout = invalidRatio > config.FALLBACK_THRESHOLD;
```

### 3. Semantic Position Fallback

If `USE_SEMANTIC_FALLBACK` enabled and threshold NOT exceeded:
- Individual nodes with (0,0,0) position try `semanticPosition` first
- Only remaining invalid nodes trigger full layout recalc

### 4. Hard Floor/Ceiling Enforcement (Backend)

**New constants in `fan_layout.py`:**
```python
MIN_Y_FLOOR = 20      # nothing below this
MAX_Y_CEILING = 5000  # reasonable upper bound
```

**Enforcement loop after anti-gravity:**
- Clamps all node Y positions to [MIN_Y_FLOOR, MAX_Y_CEILING]
- Logs violations for debugging

## Configuration Interface

```typescript
interface DevPanelConfig {
  Y_WEIGHT_TIME: number;        // 0-1
  Y_WEIGHT_KNOWLEDGE: number;   // auto = 1 - TIME
  MIN_Y_FLOOR: number;          // default 20
  MAX_Y_CEILING: number;        // default 5000
  FALLBACK_THRESHOLD: number;   // 0-1, default 0.5
  USE_SEMANTIC_FALLBACK: boolean; // default true
}
```

**Storage:** `localStorage.getItem('vetka_dev_config')`

## Event System

**Custom event emitted on Apply:**

Components can listen:
```javascript

  console.log('Config changed:', e.detail);
});
```
