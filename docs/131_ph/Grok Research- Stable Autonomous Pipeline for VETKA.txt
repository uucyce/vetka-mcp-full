## 1. Стабильность LLM провайдера (Polza 429/502 → Rock-Solid)

**Проблема:** Polza flakes → pipeline stalls. Нет backoff/fallback.

**Рекомендация:** 
- **Retry Pattern:** Exponential backoff (1s → 16s) + jitter (random ±20%) + max 5 retries.
- **Circuit Breaker:** After 3 fails/1min → pause 5min, fallback.
- **Fallback Chain:** Polza (primary) → OpenRouter → Ollama (local fallback).
- **Enforce:** Decorator на `call_model_v2` в `src/elisa/provider_registry.py`.

**Код: Retry Decorator (добавь в `provider_registry.py`):**
```python
# MARKER_131_RETRY_DECORATOR: LLM Resilience
import asyncio
import random
from functools import wraps
from typing import Callable, Any

async def resilient_call(func: Callable, *args, max_retries=5, **kwargs) -> Any:
    """Exponential backoff + jitter + fallback."""
    providers = ["polza", "openrouter", "ollama"]  # Chain order
    for provider in providers:
        kwargs["source"] = provider  # Override source
        last_exc = None
        for attempt in range(max_retries):
            try:
                return await func(*args, **kwargs)
            except Exception as e:
                last_exc = e
                if "429" in str(e) or "502" in str(e):
                    wait = (2 ** attempt) + random.uniform(-0.2, 0.2)  # Jittered backoff
                    await asyncio.sleep(wait)
                else:
                    break  # Non-retryable
        # Circuit: log + next provider
        logger.warning(f"[Circuit] Provider {provider} exhausted after {max_retries} retries")
    raise last_exc  # All failed

# Usage: Wrap calls
# result = await resilient_call(call_model_v2, messages, model="grok-4.1-fast")
```

## 3. Multi-Client Координация (Cursor/Claude → No Conflicts)

**Проблема:** Нет locking/ownership. Дубли claims.

**Рекомендация (Lightweight, No DB):**
- **Task Ownership:** C20C API + `X-Agent-ID` header + optimistic locking (check `claimed_at` timestamp + ETag).
- **File Locking:** `fcntl.flock` на open(file, 'a') handles (Python fs lock, cross-process).
- **Dedup:** Title + creator_id hash → skip if exists.

**Реализуй C20C из BRIEF (src/api/routes/task_routes.py):**
```python
# src/api/routes/task_routes.py (NEW FILE, MARKER_C20C)
from fastapi import APIRouter, Header, HTTPException
router = APIRouter(prefix="/api/tasks")

@router.post("/{task_id}/claim")
async def claim_task(task_id: str, x_agent_id: str = Header(...)):
    task = board.get_task(task_id)
    if task["status"] != "pending" or task["claimed_at"]:
        raise HTTPException(409, "Already claimed")
    board.claim_task(task_id, agent_id=x_agent_id, claimed_at=datetime.utcnow())
    return {"claimed": True, "agent": x_agent_id}

# File lock helper (в task_board.py)
import fcntl
def acquire_file_lock(file_path: str):
    with open(file_path, 'a') as f:
        fcntl.flock(f.fileno(), fcntl.LOCK_EX | fcntl.LOCK_NB)
        return f  # Hold handle during edit
```

**Для Cursor/Claude:** В `.cursorrules` или MCP: `curl -H "X-Agent-ID: cursor" /api/tasks/{id}/claim`.

## 4. Автономность Pipeline (Heartbeat + No Cascades)

**Проблема:** Polling only, cascades on bad LLM.

**Рекомендация:**
- **Heartbeat Interval:** 30s (aggressive) w/ jitter (±10s). Daemonize via C20A.
- **Event-Driven:** SocketIO emit `pipeline_complete` → wakeup (уже в approval_service.py).
- **Anti-Cascade:** Per-phase timeout (60s), checkpoint JSON save, isolate failures.
