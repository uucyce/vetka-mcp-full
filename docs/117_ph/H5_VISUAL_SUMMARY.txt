4. FLOW: How Inject_Context Works
═════════════════════════════════════════════════════════════════════════════════

    User calls: await pipeline._research(question)
                     │
                     ▼
            LLMCallTool.execute({
                "inject_context": {...}
            })
                     │
                     ▼
            _gather_inject_context()
             ├─ File 1: 8000 chars
             ├─ Session: JSON state
             ├─ Prefs: User memory
             ├─ CAM: Active nodes
             ├─ Semantic: Search results
             └─ [CHAT: Missing!]
                     │
                     ▼
            All combined + compressed
                     │
                     ▼
            <vetka_context>
            ### File: src/main.py
            ...
            ### User Preferences
            ...
            ### Semantic Search: 'auth'
            ...
            </vetka_context>
                     │
                     ▼
            Prepended to system prompt
                     │
                     ▼
            Sent to LLM (OpenRouter/Grok/Claude)
                     │
                     ▼
            Response includes semantic context
