CURRENT STATE (How It Works Today)
================================================================================

Researcher DOES get injected context:
  - Semantic search results about the research question
  - User preferences from Engram
  - All compressed with ELISION (40-60% token savings)
  - Result: Better research with less tokens

Architect DOES NOT get injected context:
  - No chat history
  - No user preferences
  - No semantic search
  - Result: Architect plans without context of chat discussions

Coder DOES NOT get injected context:
  - Planned subtask info only
  - STM (short-term memory) from previous subtasks
  - Result: Coder has some context but not chat context

================================================================================
THE OPPORTUNITY (Why H5 Matters)
================================================================================

User reports in chat:
  "Found a bug in authentication - session expires after 5 minutes"

Current behavior:
  [Architect] - Plans generic auth subtasks (doesn't see bug report)
  [Researcher] - Researches generic auth (doesn't see bug report)
  [Coder] - Implements plan without knowing about bug

Desired behavior (H5):
  [Architect] - Sees bug report, plans to fix it first
  [Researcher] - Sees bug report, researches timeout issue specifically
  [Coder] - Sees bug report, implements fix with this context

Result: Better alignment with user intent, faster bug fixes, less rework.
