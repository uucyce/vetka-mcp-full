### 1. Git Integration для AI агентов

**Лучшее решение 2025**  
**gitoxide** (https://github.com/Byron/gitoxide) — Rust-библиотека, полностью безопасная, без shell.  
+ **libgit2** (C) с bindings (nodegit, pygit2) — классика, но gitoxide быстрее и безопаснее.

**Open-source альтернатива**  
gitoxide (Rust) → Python bindings через PyO3 (или subprocess с ограничениями).

**Применимость к VETKA**  
- Агент вызывает gitoxide через subprocess или Rust binary  
- Approval flow: Socket.IO → UI prompt → user approve → commit/push  
- Ограничения: no force push, no rm -rf, branch protection (main protected)

**Код/конфиг пример** (Python + gitoxide via subprocess)

```python
# src/tools/git_tool.py
import subprocess
from pathlib import Path

def git_commit(message: str, files: list[str], branch: str = "agent-dev"):
    repo_path = Path("/path/to/vetka/repo")
    cmd = ["gitoxide", "commit", "-m", message, "--branch", branch]
    cmd.extend(files)
    
    try:
        result = subprocess.run(cmd, cwd=repo_path, capture_output=True, text=True, timeout=30)
        if result.returncode == 0:
            return {"success": True, "output": result.stdout}
        else:
            return {"success": False, "error": result.stderr}
    except subprocess.TimeoutExpired:
        return {"success": False, "error": "Git timeout"}
```

**Риски/ограничения**  
- Prompt injection → shell escape → gitoxide решает (без shell)  
- User approval — **обязательно** для push/commit

### 2. Sandboxed Code Execution

**Лучшее решение 2025**  
**E2B** (open-source версия) или **Firecracker** (AWS) + **bubblewrap** (Linux/Mac)  
Bubblewrap — самый лёгкий и быстрый для Mac (sandbox без Docker).

**Open-source альтернатива**  
- bubblewrap (https://github.com/containers/bubblewrap) — Mac/Linux  
- nsjail (https://github.com/google/nsjail) — более строгий, но сложнее на Mac  
- gVisor (Google) — тяжёлый, но безопасный

**Применимость к VETKA**  
- execute_code → bubblewrap + Python interpreter  
- Filesystem: только /tmp/sandbox + read-only project dir  
- Network: --unshare-net  
- Timeout: 30s

**Код/конфиг пример** (bubblewrap)

```python
# src/tools/execute_code.py
import subprocess

def execute_sandboxed(code: str):
    cmd = [
        "bwrap",
        "--ro-bind", "/usr/bin/python3", "/usr/bin/python3",
        "--tmpfs", "/tmp",
        "--unshare-net",
        "--die-with-parent",
        "--", "python3", "-c", code
    ]
    
    try:
        result = subprocess.run(cmd, capture_output=True, text=True, timeout=30)
        return {"stdout": result.stdout, "stderr": result.stderr, "success": result.returncode == 0}
    except subprocess.TimeoutExpired:
        return {"success": False, "error": "Execution timeout"}
```

**Риски/ограничения**  
- Mac: bubblewrap через brew, но не 100% native (лучше на Linux-сервере)  
- Approval flow — **обязательно** для exec

### 3. Browser/Terminal доступ для агентов

**Лучшее решение 2025**  
**Playwright MCP** (https://github.com/microsoft/playwright-mcp) — open-source, self-hosted  
Chrome DevTools Protocol + WebSocket → агенты видят DOM/console.

**Open-source альтернатива**  
- Playwright + Browserless (self-hosted)  
- Puppeteer + chrome-remote-interface

**Применимость к VETKA**  
- Агент запускает headless Chrome → видит console.log, DOM changes  
- Передача логов → Socket.IO → Artifact Panel (real-time)

**Код/конфиг пример** (Playwright MCP)

```python
# src/tools/browser_tool.py
from playwright.async_api import async_playwright

async def execute_browser_task(url: str, script: str):
    async with async_playwright() as p:
        browser = await p.chromium.launch(headless=True)
        page = await browser.new_page()
        await page.goto(url)
        result = await page.evaluate(script)
        console_logs = []
        page.on("console", lambda msg: console_logs.append(msg.text))
        await browser.close()
        return {"result": result, "console": console_logs}
```

**Риски/ограничения**  
- Headless Chrome → ~200–500 MB RAM  
- Network — изолировать (proxy)

### Gap 1: macOS Sandbox (альтернативы Bubblewrap)

**Bubblewrap** действительно **не работает** на macOS (Seatbelt / sandbox-exec — native, но bubblewrap — Linux-only).

**Актуальные альтернативы на Mac M-чипах (Apple Silicon)**:

| Решение                     | Уровень изоляции | Overhead (RAM/CPU) | Установка / доступность | Применимость к VETKA | Рекомендация |
|-----------------------------|-------------------|---------------------|---------------------------|-----------------------|--------------|
| **sandbox-exec** (Seatbelt) | Высокий (native macOS) | Очень низкий (~10–50 MB) | Уже встроен в macOS | **Лучший выбор** | Да — используй |
| **Colima + Docker** (VM)    | Очень высокий     | Высокий (1–2 GB RAM) | brew install colima docker | Хорошая альтернатива | Да, если нужен Docker |
| **OrbStack**                | Высокий (native VM) | Средний (~500 MB)   | brew install orbstack     | Хорошая альтернатива | Да, если нужен GUI |
| **Firejail**                | Средний           | Низкий              | brew install firejail     | Слабая (Linux-oriented) | Нет |
| **nsjail**                  | Высокий           | Низкий              | brew install nsjail       | Средняя (Linux-oriented) | Нет |

**Лучшее решение**: **sandbox-exec** (Seatbelt) — native, лёгкий, безопасный.

**Пример sandbox-exec profile** (vetka-sandbox.sb):

```plaintext
(version 1)
(allow default)

; Разрешаем только нужное
(deny file-write* file-read* process-exec*
  (subpath "/"))

(allow file-read* file-write*
  (literal "/tmp")
  (subpath "/Users/danilagulin/Documents/VETKA_Project/vetka_live_03/tmp_sandbox"))

(allow process-exec*
  (literal "/usr/bin/python3")
  (literal "/usr/local/bin/python3"))

(deny network*)

(allow mach-lookup
  (global-name "com.apple.system.logger"))
```

Запуск:

```bash
sandbox-exec -f vetka-sandbox.sb python3 -c "print('Hello from sandbox')"
```

**Overhead**:  
- RAM: +20–100 MB  
- CPU: +5–15%  
- Startup: < 50 мс

**Вывод**: Используй **sandbox-exec** — это **native macOS**, **без Docker**, **без VM**.  
Colima/OrbStack — только если нужен полный Docker.

### Gap 2: Agent Activity в 3D UI (Three.js)

**Лучшее решение 2025**  
**Custom Three.js + Socket.IO** — badge/notification + activity stream.

**Как визуализировать**:

1. **Badge/notification на узлах**  
   - При tool call (git/commit, execute_code) → добавить sprite/icon над mesh  
   - Цвета: green (success), red (error), yellow (running)  
   - Анимация: pulse scale 1.0 → 1.2

2. **Activity stream** (боковая панель или bottom bar)  
   - Socket.IO → emit('agent_activity', { agent, action, file, status })  
   - React/Vue компонент → список (timestamp, agent, icon, file path, result)

3. **Real-time updates**  
   - Socket.IO room per branch/file  
   - joinRoom(branchId) → receive updates

**Пример** (Three.js + Socket.IO):

```javascript
// В tree_renderer.py (Three.js)
socket.on('agent_activity', (data) => {
  const { branchId, agent, action, status } = data;
  const node = nodeObjects.get(branchId);
  if (node) {
    // Добавить badge
    const badge = createBadgeSprite(status);
    badge.position.set(node.position.x, node.position.y + 1.5, node.position.z);
    scene.add(badge);
    setTimeout(() => scene.remove(badge), 5000); // 5s
  }
});
```

**Риски**  
- Performance: >50 badges → WebGL overload → limit to 10–15 active

### Gap 5: Approval Flow UI

**Лучшее решение 2025**  
**Modal + diff preview** — Socket.IO + React/Vanilla modal

**Как сделать**:
- Агент вызывает tool (write_file/git_commit) → emit('approval_needed', { tool, args, diff })
- UI: modal с diff (diff2html или Monaco diff viewer)
- Кнопки: Approve / Reject / Edit

**Пример** (Vanilla JS):

```javascript
socket.on('approval_needed', (data) => {
  const modal = document.createElement('div');
  modal.innerHTML = `
    <div class="modal">
      <h3>Approve action: ${data.tool}</h3>
      <pre class="diff">${data.diff}</pre>
      <button onclick="approve(${data.id})">Approve</button>
      <button onclick="reject(${data.id})">Reject</button>
    </div>
  `;
  document.body.appendChild(modal);
});

function approve(id) {
  socket.emit('approval_response', { id, approved: true });
}
```

**Риски**  
- Timeout policy — 60–120 сек → auto-reject  
- Diff preview — git diff или Monaco (тяжёлый)
