PROBLEM IDENTIFIED:
-------------------
Users cannot see new messages from MCP agents (Claude Code, Browser Haiku) 
without reloading the browser or switching files. Messages are stored on 
the server but don't auto-sync to the client UI.

SOLUTION PROPOSED:
------------------
Add a refresh button to the ChatPanel header that:
1. Fetches latest messages from /api/chats/{chatId} (solo chat)
2. Fetches latest messages from /api/groups/{groupId}/messages (group chat)
3. Clears and reloads messages atomically
4. Disables during active typing
