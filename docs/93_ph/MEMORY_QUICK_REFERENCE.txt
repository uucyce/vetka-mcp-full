THREE CORE SYSTEMS:
===================

1. ELISIUM (Compression)
   - Location: src/memory/compression.py (504 lines)
   - Purpose: Age-based embedding compression (768D → 64D)
   - Schedule: 0d(768D) → 7d(768D) → 30d(384D) → 90d(256D) → 180d(64D)
   - Key Class: MemoryCompression, CompressedNodeState
   - Integration: JARVIS Enricher, Memory Sync, CAM Operations

2. CAM (Context-Aware Memory)
   - Location: src/orchestration/cam_engine.py (1312 lines)
   - Purpose: Dynamic tree restructuring (NeurIPS 2025 paper)
   - Operations: Branch, Prune, Merge, Accommodate
   - Key Class: VETKACAMEngine, VETKANode, CAMOperation
   - Thresholds: Novel=0.7, Merge=0.92, Prune=0.2
   - Surprise Metric: 0.0 (predictable) to 1.0 (novel)

3. ENGRAM (User Memory)
   - Location: src/memory/engram_user_memory.py (678 lines)
   - Purpose: User preferences with O(1) RAM + Qdrant
   - Levels: 1(RAM), 2(CAM+ELISION), 3(Temporal), 4(Cross-session), 5(APIs)
   - Key Class: EngramUserMemory, UserPreferences
   - Offload: After 5 accesses → promote to RAM
   - Decay: Exponential (-0.05/week)

INTEGRATION POINT:
==================
1. Scanner detects change
2. CAM engine triggered → handle_new_artifact()
3. Surprise calculated (0-1 scale)
4. Decision: Branch/Merge/Append
5. Tree restructures (Accommodation)
6. Engram updates user patterns
7. Age timer starts
8. Compression applied when age threshold reached
