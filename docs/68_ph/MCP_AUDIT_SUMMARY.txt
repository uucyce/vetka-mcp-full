================================================================================
                         CRITICAL INFORMATION
================================================================================

⚠️  SOCKET.IO IS PRIMARY MCP INTERFACE
    - MCP tools are called via Socket.IO (NOT REST)
    - Event: tool_call
    - Response: tool_result event
    - JSON-RPC 2.0 format

⚠️  APPROVAL FLOW IS ASYNCHRONOUS
    - Write operations return approval_id
    - User approves via: POST /api/approval/{id}/approve
    - Agent re-submits with _approval_id parameter
    - Tool then executes with real changes

⚠️  DRY-RUN IS SAFE BY DEFAULT
    - All write tools default to dry_run=true
    - No approval needed for dry-run
    - Use for preview and validation
    - Set dry_run=false for real changes

⚠️  RATE LIMITS ENFORCED
    - Hitting limit returns: error: rate_limited
    - Includes retry_after field (seconds)
    - Apply per user/client
