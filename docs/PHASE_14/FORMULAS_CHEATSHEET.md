## 🔢 FORMULA 1: LAYER_HEIGHT

### Математическая запись:
```
layer_height = (screen_height × 0.8 / max_depth) × file_factor

где:
  file_factor = 1 + (estimated_files × 0.3 / 50)
```

### Входные данные:
| Параметр | Пример | Описание |
|----------|--------|----------|
| `screen_height` | 1080 | Высота экрана в пикселях |
| `max_depth` | 4 | Максимальная глубина дерева |
| `max_files_per_folder` | 50 | Макс файлов в одной папке |

### Расчёт (шаг за шагом):

**МАЛЕНЬКОЕ ДЕРЕВО:**
```
Входные: screen_height=1080, max_depth=3, max_files=10

Шаг 1 (base):        1080 × 0.8 / 3 = 288
Шаг 2 (file_factor): 1 + (8 × 0.3 / 50) = 1.048
Шаг 3 (hasil):       288 × 1.048 = 302
Constraint:          max(60, min(300, 302)) = 300px

📌 РЕЗУЛЬТАТ: 300px (комфортный слой)
```

**БОЛЬШОЕ ДЕРЕВО:**
```
Входные: screen_height=1080, max_depth=6, max_files=100

Шаг 1 (base):        1080 × 0.8 / 6 = 144
Шаг 2 (file_factor): 1 + (80 × 0.3 / 50) = 1.48
Шаг 3 (hasil):       144 × 1.48 = 213
Constraint:          max(60, min(300, 213)) = 213px

📌 РЕЗУЛЬТАТ: 213px (сжатый, но видимый слой)
```

### Формула объяснена:
- **0.8** = используем только 80% экрана (20% на margins)
- **/ max_depth** = распределяем равномерно по слоям
- **× file_factor** = если файлов много, добавляем место
- **max(60, min(300))** = constraint: минимум 60px (читаемо), максимум 300px (не пусто)

### Результат:
- ✅ Маленькие деревья = широкие слои (удобно)
- ✅ Большие деревья = узкие слои (но видно)
- ✅ ВСЕГДА видно всё дерево

## 🔢 FORMULA 2: LAYER_SPACING

### Математическая запись:
```
spacing = (screen_width - 2 × margin) / (max_folders - 1)

constraint: max(60, min(400, spacing))
```

### Входные данные:
| Параметр | Пример | Описание |
|----------|--------|----------|
| `screen_width` | 1920 | Ширина экрана в пикселях |
| `max_folders_in_layer` | 5 | Макс папок в одном слое |
| `margin` | 200 | Зазор с краёв экрана |

### Расчёт (шаг за шагом):

**МАЛО ПАПОК (2):**
```
Входные: screen_width=1920, max_folders=2, margin=200

Шаг 1 (available):  1920 - 2×200 = 1520
Шаг 2 (spacing):    1520 / (2-1) = 1520
Constraint:         max(60, min(400, 1520)) = 400px

📌 РЕЗУЛЬТАТ: 400px (папки ДА-А-АЛЕКО друг от друга)
```

**СРЕДНЕЕ КОЛИЧЕСТВО ПАПОК (5):**
```
Входные: screen_width=1920, max_folders=5, margin=200

Шаг 1 (available):  1920 - 2×200 = 1520
Шаг 2 (spacing):    1520 / (5-1) = 380
Constraint:         max(60, min(400, 380)) = 380px

📌 РЕЗУЛЬТАТ: 380px (папки нормально распределены)
```

**МНОГО ПАПОК (20):**
```
Входные: screen_width=1920, max_folders=20, margin=200

Шаг 1 (available):  1920 - 2×200 = 1520
Шаг 2 (spacing):    1520 / (20-1) = 80
Constraint:         max(60, min(400, 80)) = 80px

📌 РЕЗУЛЬТАТ: 80px (папки близко, но всё видно на экран)
```

### Формула объяснена:
- **screen_width - 2×margin** = свободное место для папок
- **/ (max_folders - 1)** = равномерное распределение
- **max(60, min(400))** = constraint: 60-400px (читаемо)

### Результат:
- ✅ 2 папки → далеко (400px)
- ✅ 5 папок → нормально (380px)
- ✅ 20 папок → узко (80px)
- ✅ ВСЕГДА видны ВСЕ папки

## 🔢 FORMULA 3: FILE_SPACING

### Математическая запись:
```
spacing = (layer_height × 0.7) / files_count

constraint: max(30, min(60, spacing))
```

### Входные данные:
| Параметр | Пример | Описание |
|----------|--------|----------|
| `files_count` | 20 | Количество файлов в папке |
| `layer_height` | 200 | Высота слоя (из FORMULA 1) |

### Расчёт (шаг за шагом):

**МАЛО ФАЙЛОВ (3):**
```
Входные: layer_height=200, files_count=3

Шаг 1 (available):  200 × 0.7 = 140
Шаг 2 (spacing):    140 / 3 = 47
Constraint:         max(30, min(60, 47)) = 47px

📌 РЕЗУЛЬТАТ: 47px (файлы КОМФОРТНО, легко читать)
```

**СРЕДНЕЕ КОЛИЧЕСТВО ФАЙЛОВ (10):**
```
Входные: layer_height=200, files_count=10

Шаг 1 (available):  200 × 0.7 = 140
Шаг 2 (spacing):    140 / 10 = 14
Constraint:         max(30, min(60, 14)) = 30px

📌 РЕЗУЛЬТАТ: 30px (файлы сжаты, но видны)
```

**МНОГО ФАЙЛОВ (50):**
```
Входные: layer_height=100, files_count=50

Шаг 1 (available):  100 × 0.7 = 70
Шаг 2 (spacing):    70 / 50 = 1.4
Constraint:         max(30, min(60, 1.4)) = 30px

📌 РЕЗУЛЬТАТ: 30px (файлы максимально сжаты, но видны)
```

### Формула объяснена:
- **layer_height × 0.7** = 70% на файлы, 30% на папку
- **/ files_count** = равномерное распределение
- **max(30, min(60))** = constraint: 30-60px (минимум видимости)

### Результат:
- ✅ 3 файла → 47px (комфортно)
- ✅ 10 файлов → 30px (сжато, но видно)
- ✅ 50 файлов → 30px (макс сжато, но видно)
- ✅ НИКОГДА не перекрываются

## 🎯 ИТОГОВАЯ ТАБЛИЦА

| Что | Формула | Зависит от | Результат |
|-----|---------|------------|-----------|
| **LAYER_HEIGHT** | `(SH×0.8 / D) × FF` | Глубины + Файлов | Слои распределены правильно |
| **LAYER_SPACING** | `(W - 2M) / (F-1)` | Папок в слое | Папки видны все |
| **FILE_SPACING** | `(LH × 0.7) / FC` | Файлов в папке | Файлы не перекрываются |

**Легенда:**
- SH = screen_height
- D = max_depth
- FF = file_factor
- W = screen_width
- M = margin
- F = max_folders
- LH = layer_height
- FC = files_count

## ✅ ПРОВЕРКА

Знаешь ли ты что такое формула теперь?

✅ **ФОРМУЛА** = математическое выражение с переменными  
Например: `y = 2x + 3` или `layer_height = (SH × 0.8 / D) × FF`

❌ **НЕ ФОРМУЛА** = просто описание  
Например: "зависит от глубины и файлов"
