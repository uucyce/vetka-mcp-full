### Chapter 44: Slip, Slide, Ripple, Roll Edits (p.693-712) — GOLD

**FCP7 defines 4 essential trim edit types. CUT has ZERO.**

| FCP7 Trim Tool | Key | What It Does | CUT Status | Gap |
|---|---|---|---|---|
| **Ripple Edit** | RR | Trims one side, shifts everything after | MISSING | CRITICAL |
| **Roll Edit** | R | Moves edit point, doesn't change duration | MISSING | CRITICAL |
| **Slip Edit** | S (×2) | Changes content within clip without moving it | MISSING | CRITICAL |
| **Slide Edit** | S (×3) | Moves clip between neighbors, adjusts their durations | MISSING | CRITICAL |

**FCP7 Trim capabilities in detail:**
- Ripple: Extends/shortens clip In/Out point, all subsequent clips shift
- Roll: Adjusts edit point between two clips, total duration unchanged
- Slip: Changes the In/Out within media handle, clip position unchanged
- Slide: Moves clip in timeline, adjacent clips adjust to fill

**CUT current trim:** Only basic edge drag in TimelineTrackView.tsx (hard cut resize). No tool selection, no two-up display, no asymmetric trim.

**Required components:**
1. Tool state machine: `activeTool` in store (select/razor/ripple/roll/slip/slide/hand/zoom)
2. Cursor changes per tool
3. Two-up display in Canvas during trim (FCP7 shows outgoing/incoming frames)
4. Trim Edit Window (Ch.45-46) — dedicated window for precision trimming

## Architectural Recommendations

### 1. Tool State Machine (HIGHEST PRIORITY)
FCP7's entire editing paradigm depends on tool selection. Without it, trim tools and many editing operations are impossible.

```
Store: activeTool: 'select' | 'razor' | 'ripple' | 'roll' | 'slip' | 'slide' | 'hand' | 'zoom' | 'speed'
Keys: A(select), B(blade), R(ripple→press again→roll), S(slip→press again→slide), SSS(speed)
Cursor: Changes per tool
Click behavior: Depends on activeTool + what was clicked (clip body, edit point, etc.)
```

### 2. Source/Program Monitor Split (BLOCKS Match Frame)
Currently both monitors show same feed. FCP7 Viewer (Source) and Canvas (Program) are independent.
This blocks: Match Frame, Insert/Overwrite, panel-scoped I/O marks.

### 3. Panel Focus System (BLOCKS panel-scoped hotkeys)
FCP7's JKL, I/O, Delete all depend on which window has focus.
