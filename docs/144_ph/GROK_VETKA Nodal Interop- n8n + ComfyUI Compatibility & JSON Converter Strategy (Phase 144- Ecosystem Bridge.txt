## 🎯 **Почему это работает? (Совместимость Breakdown)**
Все три системы — **node-based graphs** с **JSON сериализацией**:

| System | JSON Structure | Nodes | Edges/Links | VETKA Mapping |
|--------|----------------|-------|-------------|---------------|
| **n8n** | `Array<{id, type, position, parameters, connections: {outputs: {...}}}}` | `type: "n8n-nodes-base.httpRequest"` etc. | `connections` (multi-output) | DAGNode.id/label/type → node.id/type; DAGEdge.source/target → connections |
| **ComfyUI** | `{nodes: {id: {class_type, inputs, outputs, ...}}, links: [{from_node, to_node, ...}]}` | `class_type: "KSampler"` (AI ops) | `links[]` array | DAGNode → nodes[id]; DAGEdge → links (strength → weight) |
| **VETKA (DAG)** | `DAGNode[] + DAGEdge[]` (из MCC: id, type, status, layer, taskId; source/target/strength) | taskId/role/model | strength/type | **Native!** Уже в MyceliumCommandCenter (mapBackendNode/Edge) |

- **Общий denominator**: **JSON Graph Schema** (nodes + edges/links). ~90% **direct mapping** без потери (positions/layers → optional).
- **Преимущества**:
  - **Импорт**: Топ n8n workflows (HTTP→LLM chains, 10k+ community) → VETKA DAG для 3D viewport + mycelium_auditor.py research.
  - **Экспорт**: VETKA Mycelium DAG (Phase 143 MCC) → ComfyUI для AI gen (e.g., viewport screenshots → SD workflows).
  - **Mutual**: n8n/Comfy load VETKA JSON via converter → их UI; обратно — через API.

**Лимиты/Риски**: 
- n8n: Free OSS, но cloud — $20+/mo (self-host unlimited). JSON import via CLI/API.
- ComfyUI: Free, JSON hot-load. Custom nodes → fallback to "generic" в VETKA.
- Конверсия: ~95% success; edge cases (n8n credentials) → prompt user.
