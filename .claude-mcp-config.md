## API Endpoints

### 1. Read Messages
**GET** `/api/debug/mcp/groups/{group_id}/messages?limit=10`

Get the latest messages from the group chat.

**Example:**
```javascript
fetch('http://localhost:5001/api/debug/mcp/groups/542444da-fcb1-4e26-ac00-f414e2c43591/messages?limit=10')
  .then(response => response.json())
  .then(messages => {
    console.log('Latest messages:', messages);
  });
```

**Query Parameters:**
- `limit` (optional): Number of messages to retrieve (default: 10)

---

### 2. Send Message
**POST** `/api/debug/mcp/groups/{group_id}/send`

Send a message to the group chat.

**Request Body:**
```json
{
  "agent_id": "browser-haiku",
  "sender": "Browser Haiku",
  "content": "Your message here"
}
```

**Example:**
```javascript
fetch('http://localhost:5001/api/debug/mcp/groups/542444da-fcb1-4e26-ac00-f414e2c43591/send', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    agent_id: 'browser-haiku',
    sender: 'Browser Haiku',
    content: 'Hello from Browser Haiku!'
  })
})
  .then(response => response.json())
  .then(result => {
    console.log('Message sent:', result);
  });
```

---

### 3. Check Pending Messages
**GET** `/api/debug/mcp/pending/{agent_id}`

Check if there are any pending messages for you.

**Example:**
```javascript
fetch('http://localhost:5001/api/debug/mcp/pending/browser-haiku')
  .then(response => response.json())
  .then(pending => {
    console.log('Pending messages:', pending);
  });
```

## Complete JavaScript Integration Example

### Simple Chat Client
```javascript
class VETKAChatClient {
  constructor() {
    this.baseURL = 'http://localhost:5001';
    this.groupId = '542444da-fcb1-4e26-ac00-f414e2c43591';
    this.agentId = 'browser-haiku';
    this.sender = 'Browser Haiku';
  }

  // Read latest messages
  async getMessages(limit = 10) {
    const url = `${this.baseURL}/api/debug/mcp/groups/${this.groupId}/messages?limit=${limit}`;
    const response = await fetch(url);
    return await response.json();
  }

  // Send a message
  async sendMessage(content) {
    const url = `${this.baseURL}/api/debug/mcp/groups/${this.groupId}/send`;
    const response = await fetch(url, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({
        agent_id: this.agentId,
        sender: this.sender,
        content: content
      })
    });
    return await response.json();
  }

  // Check pending messages
  async checkPending() {
    const url = `${this.baseURL}/api/debug/mcp/pending/${this.agentId}`;
    const response = await fetch(url);
    return await response.json();
  }

  // Display messages in console
  async displayMessages() {
    const messages = await this.getMessages();
    console.log('=== VETKA MCP Chat ===');
    messages.forEach(msg => {
      console.log(`[${msg.timestamp}] ${msg.sender}: ${msg.content}`);
    });
  }

  // Poll for new messages every N seconds
  startPolling(intervalSeconds = 5) {
    setInterval(async () => {
      const pending = await this.checkPending();
      if (pending.has_pending) {
        console.log('New messages available!');
        await this.displayMessages();
      }
    }, intervalSeconds * 1000);
  }
}

// Usage
const chat = new VETKAChatClient();

// Read messages
await chat.getMessages(10);

// Send a message
await chat.sendMessage('Hello team! Browser Haiku reporting for duty.');

// Check for pending messages
await chat.checkPending();

// Display all messages nicely
await chat.displayMessages();

// Start auto-polling for new messages
chat.startPolling(5); // Check every 5 seconds
```

## Usage Patterns

### Pattern 1: Check and Respond
```javascript
// Check for messages mentioning you
const messages = await chat.getMessages(20);
const mentionsMe = messages.filter(m =>
  m.content.toLowerCase().includes('browser') ||
  m.content.toLowerCase().includes('haiku')
);

if (mentionsMe.length > 0) {
  console.log('You were mentioned:', mentionsMe);
  // Respond to the most recent mention
  await chat.sendMessage('Responding to your request...');
}
```

### Pattern 2: Status Updates
```javascript
// Send status update after completing a task
async function reportTaskCompletion(taskName, result) {
  await chat.sendMessage(
    `✓ Task completed: ${taskName}\nResult: ${result}`
  );
}

// Usage
await reportTaskCompletion('Navigate to VETKA dashboard', 'Success - loaded in 2.3s');
```

### Pattern 3: Request Help
```javascript
// Ask the team for help
async function requestHelp(issue) {
  await chat.sendMessage(
    `⚠️ Browser Haiku needs help:\n${issue}\n\n@PM @Architect`
  );
}

// Usage
await requestHelp('Cannot find the submit button on the form. CSS selector needed.');
```

## Troubleshooting

### Server Not Responding
```javascript
// Check if server is running
fetch('http://localhost:5001/health')
  .then(r => console.log('Server is running'))
  .catch(e => console.error('Server is down. Start with: python src/vectorizer/mcp_server.py'));
```

### Messages Not Sending
- Verify you're using the correct `agent_id`: `browser-haiku`
- Check that the request body includes all required fields
- Ensure Content-Type header is set to `application/json`

### Not Receiving Messages
- Use the pending endpoint to check for new messages
- Increase the `limit` parameter when fetching messages
- Check the timestamp of the last message you received
