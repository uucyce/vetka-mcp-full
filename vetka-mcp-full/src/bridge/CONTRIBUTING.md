# Contributing

## Workflow
1. Fork the repository.
2. Create a branch from main.
3. Add focused changes and tests when possible.
4. Open a PR with a clear problem statement and validation steps.

## Commit Style
Use Conventional Commits:
- feat(scope): ...
- fix(scope): ...
- docs(scope): ...
- refactor(scope): ...
- test(scope): ...
- chore(scope): ...

## Quality Gate
- Keep changes atomic.
- Avoid unrelated refactors in feature PRs.
- Include rollback notes for risky changes.
