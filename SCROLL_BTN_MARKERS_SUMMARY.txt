ЛОГИКА STATE-а
════════════════════════════════════════════════════════════════════════════════

  Formula: scrollHeight - scrollTop - clientHeight < 50px

  Пример 1 (ВНИЗУ):
  ─────────────────
    scrollHeight = 1000px (всего контента)
    scrollTop = 400px (как далеко скроллили)
    clientHeight = 600px (видимая область)
    Distance = 1000 - 400 - 600 = 0px
    0 < 50? YES → isAtBottom = true → Кнопка СКРЫТА

  Пример 2 (ВЫШЕ):
  ─────────────────
    scrollHeight = 1000px
    scrollTop = 100px (скроллили выше)
    clientHeight = 600px
    Distance = 1000 - 100 - 600 = 300px
    300 < 50? NO → isAtBottom = false → Кнопка ВИДНА (↓)

  Threshold: 50px (пользователь может быть до 50px выше, кнопка всё ещё скрыта)


REFS ИСПОЛЬЗУЮТСЯ
════════════════════════════════════════════════════════════════════════════════

  messagesContainerRef:
    - Type: useRef<HTMLDivElement>(null)
    - Target: <div overflow:auto> (строка 2242)
    - Usage:
      * Scroll listener: container.addEventListener('scroll', handleScroll)
      * Height calculations: container.scrollHeight, scrollTop, clientHeight
      * Auto-scroll: container.scrollTop = 0 (если добавить в будущем)

  messagesEndRef:
    - Type: useRef<HTMLDivElement>(null)
    - Target: <div ref={messagesEndRef} /> (строка 2255)
    - Usage:
      * Auto-scroll: messagesEndRef.current?.scrollIntoView({ smooth })
      * Маркер конца списка сообщений
      * Невидимый, служит только как целевая позиция


AUTO-SCROLL ПРИ НОВЫХ СООБЩЕНИЯХ
════════════════════════════════════════════════════════════════════════════════

  useEffect(() => {
    setTimeout(() => {
      messagesEndRef.current?.scrollIntoView({ behavior: 'smooth' });
    }, 0);
  }, [chatMessages.length]);

  Когда срабатывает: Когда chatMessages.length изменился (новое сообщение)
  Что делает:
    - setTimeout(..., 0) → дать DOM обновиться асинхронно
    - scrollIntoView() → прокручивает контейнер к messagesEndRef
    - behavior: 'smooth' → плавная анимация (~200-300ms)

  Результат: Новое сообщение автоматически видно пользователю


СОБЫТИЕ SCROLL
════════════════════════════════════════════════════════════════════════════════

  useEffect(() => {
    const container = messagesContainerRef.current;
    if (container) {
      container.addEventListener('scroll', handleScroll);
      return () => container.removeEventListener('scroll', handleScroll);
    }
  }, [handleScroll]);

  Что происходит:
    1. На каждый scroll event → вызывается handleScroll()
    2. Вычисляется расстояние до конца
    3. Обновляется isAtBottom (true/false)
    4. Conditional render: {!isAtBottom && (<button>)} → кнопка показывается/скрывается
    5. При unmount → удаляется listener (cleanup)
